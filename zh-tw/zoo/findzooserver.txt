======Rhino 5如何尋找Zoo伺服器主機？======
> **產品：** //[[zh-tw:zoo:zoo5|Zoo 5]]//
> **摘要：** //Rhino 5如何尋找Zoo伺服器管理主機概述。//

=====概述=====
不像Rhino 4.0可以自動在同個網域內尋找Zoo 4.0伺服器主機,Rhino 5需要知道Zoo 5伺服器主機名稱。

當Rhino 5要從Zoo 5取得一個授權，會依照以下順序確認查找Zoo 5伺服器主機名稱：
  - 查找本地電腦的Windows註冊表主機名稱或您伺服器主機IP位置。
  - 查找目前使用者的Windows註冊表主機名稱或您伺服器主機IP位置。
  - 查詢您的網域名稱系統(DNS)伺服器為預設Zoo伺服器主機名稱。
 

=====更多訊息=====

====註冊表查找-目前使用者====
當您查找登錄編輯程式HKEY_CURRENT_USER的Zoo server名稱或IP位置。Rhino 5 32及64位元系統將到以下位置尋找：

<code>
Hive:  HKEY_CURRENT_USER
Key:   Software\McNeel\Rhinoceros\5.0\License Manager
Name:  Server
Type:  REG_SZ
Value: <host name or IP address>
</code>

====註冊表查找-目前使用者====

若您查找登錄編輯程式HKEY_CURRENT_USE的Zoo server名稱或IP位置。Rhino 32及64位元版本皆會尋找這個位置：

<code>
Hive:  HKEY_CURRENT_USER
Key:   Software\McNeel\Rhinoceros\5.0\License Manager
Name:  Server
Type:  REG_SZ
Value: <host name or IP address>
</code>

====使用DNS====
若Rhino 5無法通過登錄編輯程式確認您的Zoo 5伺服器主機名稱，則會查詢您的DNS伺服器名稱為Zoo預設名稱，以下則是主機名稱：

<code>
__mcneel.__zoo5
</code>

舉例，若貴公司主機名稱為"mycorp.com"，那摩您可以增加一個DNS別名為<nowiki>"__mcneel.__zoo5.mycorp.com"</nowiki>用來指向您的Zoo伺服器名稱。
