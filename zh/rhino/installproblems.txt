  - 当有错误发生时，会跳出有一个连结的对话框。 
  - 点选连结并开启log并请存档成一个txt文件。
  - 请将此文件email到[[tech@mcneel.com]]

-->在新计算机上安装最新更新版本#
**//问：//** //我买了一台新计算机并且想安装Rhino版本**X** SR **Y**。我有购买升级版本的Rhino，我需要安装所有的旧版及修正版吗？ //

**//答：//**  您只要安装您最近版本的安装光盘。在安装时，将会检查是否会有旧版，您只要将光盘片放入检查，并不需要实际安装。当您安装后，检查并升级到最新的修正版即可。(Rhino 5则是会要求您输入旧版序号)\\

<--

-->什么是修正版？#

修正版是针对现存版本进行错误修正，维护性更新，并且为免费提供。这些安装程序可经由网络下载、并可安装在已安装程序的计算机上安装。修正版可由两种方式确认版本，一是SR编号，例如：SR3 或 SR4b，另一为日期，可由下载文件名称上看到例如：20050729，则代表是公元2005年7月29日版本。
<--

-->如何知道我目前安装的版本为？#
您可由程序中的说明>关于查看您的软件程序版次，会出现Rhino开启的画面，您可以由”关于”了解目前的版次。 [[rhino:rhinoceros|Rhinoceros]].
\\
若要查Flamingo的版次，请在指令列输入 FlamingoAbout 指令，Bongo则输入BongoAbout 指令。

<--

-->如何取得修正版？#

您可以在Rhino的说明>检查是否有新版本来快速了解是否有新的修正版。若您的计算机有连接网络，McNeel的网络服务器将会提供您最新版的下载连结。若您使用Flamingo，请在Raytrace或Photometric下找检查是否有新版本。若您使用Bongo，请到Bongo>找检查是否有更新。
\\
在Rhino 5，请到**工具>选项>更新与统计**并设定更新频率为"修正版" 或者是您可以尝试改到"修正版候选版"。 当您关闭Rhino，您将会收到安装最新修正版的提示，您可以选择是否要立即安装。

<--

-->我可以在卸载与修改程序这边直接卸载较旧的版本且不会有任何问题？#
您在系统控制台的卸载与修改程序可以看到所有已安装的修正版本，您可以发现每个更新版本皆使用约250MB的硬盘空间，这部分有些误导，微软系统回报所有已安装的档案被添加到现有的安装文件中。请**不要**任意自行移除较旧版本的Rhino 5修正版本。

-->Rhino 5.0安装不完整，缺少语言包#
您是否缺少了模板档、工具列或外挂程序？这些是语言包没有安装的征状，这可能是某些安装过程中产生的错误或不完整安装。

若您在安装过程中缺少了些语言包，您可以由Rhino 5安装光盘中查找Installer文件夹，您将需要手动自行完成安装。

若要确认是否有安装好语言支持给插件及界面，您可以找寻下列路径：
C:\Users\<login>\AppData\Roaming\McNeel\Rhinoceros\5.0. 

若资料夹不见或您已经开启Rhino，您将会发现没有工具列、模板档及说明档。
请依照下列步骤修复：
     - 执行光盘中的繁体中文版的LuanguaPack_zh-tw.msi档案。
     - 开启Rhino并输入**ToolbarReset**。
     - 关闭Rhino并重新开启。这将会将遗漏的部分都重新设定。

-->Error 1327 - Invalid Drive:#
**//问：//** //安装时有一个错误提示为 "Error 1327. Invalid Drive (E:, F:, etc)"的错误 //

**//产生原因：//** //您的使用者账户设定有几个虚拟盘用来指定为特定资料匣，例如”我的文件” //

**//解决方式：//**
  - 微软的开始键
  - 输入 RegEdit 输入enter
  - 寻找下列路径HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Shell Folders
  - 由安装档案寻找错误提示错误的资料盘，变更至有效的盘或者资料夹即可。  - 寻找下列路径 HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders
  - Look for directories listed that contain the drive letter mentioned in the error from 

**注意：** 这个变更登录资料的解决方式，若操作不正却很可能会造成您计算机严重的问题。因此，请您务必小心执行这些步骤。此外，您可以在修改之前，先备份注册表。如果发生问题，您可以还原回复。关于备份及还原注册表的更多提示，您可以参考下列微软技术支持文章：
[[http://support.microsoft.com/kb/322756|如何在 Windows XP 中备份及还原注册表]]

-->Error 1402 or 1406 (Could not open key: UNKNOWN\Components)#
** //问题：//** //若您看到错误提示 "Could not open key: UNKNOWN\Components\(lots of letters and numbers).Error 1402. 确认您有足够权限，请联络您的计算机管理员"//

**//造成原因//** 一个或多个注册码无法写入 HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Installer\UserData\S-1-5-<numbers>\Components没有足够权限。

**//解决方案1：//**
在微软的注册表，在搜寻栏输入"CMD"。在"cmd.exe"上按右键然后选择"以使用者管理员权限执行"。复制下列内容并复制至cmd，按Enter。然后Rhino将可安装。

<code>
secedit /configure /cfg %windir%\inf\defltbase.inf /db defltbase.sdb /verbose
</code>

**//解决方案2：//**
//警告：下列方式将编辑微软机码，若有错误可能会造成计算机无法开机，若您担心有相关问题产生，请直接联络McNeel技术人员服务。//
  - 由微软的TechNet下载 [[http://technet.microsoft.com/en-us/sysinternals/bb897553|psexec]]并解压缩放在计算机桌面。
  - 开启命令提示字符(Windows command shell)
  - 将路径改到您的桌面
  - 在指令列输入： **psexec -i -d -s c:\windows\regedit.exe**这会导致 Windows 运行登录编辑程序使用系统账户将有比账户管理员更强大的权限。
  - 浏览HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Installer\UserData\
  - 在**UserData**资料夹上按右键，然后点选**Permissions...**
  - 在Permissions对话框选择 **Advanced**
  - 确认在这个资料夹"系统管理者身分(Administrators)"有"完全控制" 权限，并且设定使用者有可读的权限。
  - 选择 **Replace all child object permissions with inheritable permissions from this object**
  - 点选两次 **OK**以套用新的权限设定。
  - 再次安装Rhino。

-->Error 1603#

造成1603错误的情形有很多可能性，请依照下列步骤做做看。

\\
\\
**步骤 1：安装.NET 4.0 Framework**
  - 开启**档案管理员**并且找寻Rhino安装DVD光盘
  - 开启**Installers\Prerequisites**资料夹
  - 执行[[http://www.microsoft.com/en-us/download/details.aspx?id=17718|dotNetFx40_Full_x86_x64.exe]].
  - 若Microsoft .NET 安装程序**出现错误提示**： 
    - 抄写下错误信息 
    - **进行步骤2**
  - 仅限64-bit Windows XP：
    - 下载并安装[[http://www.microsoft.com/en-us/download/details.aspx?id=8483|Windows Installer 4.5 Redistributable]]
  - **跳过步骤3**
\\
**步骤2：更新Windows**
  - 请[[http://windows.microsoft.com/en-US/windows/help/windows-update|更新微软操作系统]]
  - 若安装并更新所有重要的更新。这部分可能会花很长的时间作业。
  - 若您在更新的时候有错误，请看此篇关于 [[http://support.microsoft.com/kb/906602|微软更新常见问题]]文章
  - 当您更新完[[http://windows.microsoft.com/en-US/windows/help/windows-update|Windows Update service]]并解决微软更新问题 ，请**重新再做一次步骤1**.
\\
**步骤 3:安装 Rhino**
  - 开启**文件管理**
  - 找到Rhino 5 DVD安装光盘
  - 试着执行DVD路径下的**Setup.exe**
  - 若安装仍失败，请将错误log文件存在您的桌面。
  - **进行步骤 4**
\\
**步骤 4：手动安装**
  - 开启**文件管理**
  - 找Rhino 5 DVD安装光盘下的**Installers**资料夹
  - 若计算机是 //64位作业系统//：双击**rhino5setup_en-us_x64.msi**并完成安装n
  - 给所有系统： 
      - 双击**rhino5setup_en-us_x86.msi**并完成安装
      - 双击**LanguagePack_en-us.msi** (及其他任何您希望安装的语言版本，简体中文为nguagePack_zh-cn.msi)
      - 双击**HelpMedia.msi**
  - 若您在安装过程中有遇到任何错误：
    - 抄写下错误代码及错误提示
    - 将您进行的安装步骤及错误提示email至tech.cn@mcneel.com
  - 若需要，请重新开启Windows
  - **开启 Rhino 5**
  - 在Rhino 5指令列，输入**ToolbarReset**并按Enter键
  - **关闭 Rhino 5**
  - **开启 Rhino 5，确认工具列皆正常**
  - **安装完成**
\\
**仍然都不无法成功安装？**
  - 我们很抱歉，真是辛苦您了！
  - 若有错误log文件，请email至tech.cn@mcneel.com

-->Rhino 5 安装程序一直要求安装.NET Framework 4#
**问：**\\ Rhino 5 安装程序一直要求安装.NET Framework 4并且Rhino 5安装失败，但.NET Framework 4 实际已安装。

**原因：**\\ 系统中的.NET Framework 4不知明原因已损坏。

**解决方式：**
  - 下载<color #999999>[[http://files.na.mcneel.com/misc/dotnetfx_cleanup_tool.zip|.NET Framework 卸载工具]]</color>.
  - 执行 .NET Framework 卸载工具
  - 选择 **.NET Framework - All versions** 移除。\\ {{http://wiki.mcneel.com/_media/rhino/rhino5/k01521.png?w=405}}
  - 选择 **Cleanup Now** 按钮。
  - 重新启动计算机。
  - 使用Rhino 5 DVD 安装光盘安装Rhino 5。\\ <color #999999> Rhino 5 DVD安装光盘将会先安装.NET Framework 4 再安装Rhino 5。\\ 若你想安装</color><color black>Rhino 5 评估版本</color><color #999999>，您需要先 [[http://www.microsoft.com/en-us/download/details.aspx?id=17718|下载]] and re-install .NET Framework 4 full version before installing Rhino 5. </color>

-->安装程序不支持Windows 7/8？#

**问：**\\ 我目前使用Windows 7/8，但我在使用Rhino安装光盘安装时系统却提示不支持？

**答：**\\ 
  - 在setup.exe文件上按右键，选择**内容**。
  - 到**兼容性**标签。\\ <color #999999>此兼容性模式可能被设定到"Windows XP (Service Pack 2)"且无法更改。</color>
  - 选择对话框下方的**改变所有使用者设定**按钮。
  - 将 **以兼容模式执行这个程序**取消勾选。 
  - 按**确定**两次结束此对话框。
  - 再次在exe安装程序上按右键并选择**以系统管理员身分执行**安装。
