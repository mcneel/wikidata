======Rhino 4에서 내 도구모음이 사라졌어요!======

[[toolbarsdissappearv5|**Rhino 5**에서 도구모음이 사라지는 경우에는 이 페이지를 참조하세요]]

**Rhino 4의 도구모음**은 .tb 확장자를 가진 외부 파일에 저장되어 있습니다. 이 파일은 때때로 작업 영역 파일이라고도 불립니다. Rhino를 켰을 때 도구모음이 사라졌다면 대부분의 경우 Rhino 에서 도구모음 파일로의 경로를 "잃어버렸기" 때문입니다. 원인은 여러 가지일 수 있습니다. 다음 방법으로 도구모음을 복원하세요:

//**먼저**//,  기본 작업 영역 (default.tb) 을 사용한다면, **ToolbarReset** 명령을 실행하시면 됩니다. 이 방법으로 문제가 바로 해결되시기를 바랍니다.

> **안내:** default.tb 파일을 수정했다면, 원래 파일이 아니라 수정된 도구모음이 로드됩니다.

**사용자 지정 작업 영역을 사용하고 있다면** (다른 파일 이름으로 저장하여 사용해온 작업 영역), ToolbarReset 명령은 default.tb를 다시 로드하므로 원하시는 작업이 실행되지 않습니다.

어떤 이유에서건 ToolbarReset 명령으로 원하는 결과를 얻지 못 할 때, 다음 과정을 실행해 보세요:

**도구 > 도구모음 레이아웃**으로 가서 **파일 > 열기**를 선택합니다.

열린 대화상자에서 원하시는 도구모음 파일이 안 보인다면(기본 도구모음 레이아웃을 사용하는 경우에는 default.tb가 해당됩니다), 다음 폴더를 찾아보세요: 

  * Windows Vista, 7 또는 8: C:\ProgramData\McNeel\Rhinoceros\4.0
  * Windows XP: C:\Documents and Settings\All Users\Application Data\McNeel\Rhinoceros\4.0

해당 경로에 //반드시// default.tb 를 찾으실 수 있으며, 이 경로에 사용자 지정한 작업 영역을 저장했다면 해당 파일도 찾으실 수 있을 것입니다.

이후에도 계속해서 작업 영역이 열리지 않으면 해당 파일이 손상되었을 수도 있습니다. 사용하고 있는 도구모음이, 수정하지 않은 기본 작업 영역 (default.tb) 이라면, 같은 디렉터리에 default.tb.bak 파일을 찾을 수 있을 것입니다. Rhino 의 모든 창을 닫고, 손상된 파일과 해당 파일의 .bak 파일을 모두 제거한 후 다시 시도해 보세요.

여전히 문제가 계속되면, default_backup.tb 파일을 사용하는 응급 처치 방법이 있습니다. 이 파일의 복사본을 만들고 이름을 default.tb 로 지정합니다.

사용자 지정 작업 영역을 사용 중이고, 사용 중에 Rhino를 한 번이라도 열고 닫았었다면 .bak 파일도 있을 것입니다. 해당 .bak 파일이 보이지 않는다면, 다른 위치에서 사용자 지정 작업 영역의 백업 복사본을 찾아두셔야 합니다. 백업 복사본은 만들어두셨죠?

> **안내:** 기본 작업 영역(**default.tb**)을 어떻게든 수정했다면 수정한 내역은 새로운 이름으로 저장해두시는 것을 권장합니다. Rhino를 새로 설치하거나 업데이트하면 **default.tb** 파일이 덮어쓰기되어, 기존에 수정해둔 **모든** 사용자 지정 설정 내역이 지워집니다. **도구 > 도구모음 레이아웃**으로 가서, **파일 > 다른 이름으로 저장**을 눌러, 수정된 작업 영역을 새로운 이름으로 저장합니다. 사용자 지정한 작업 영역을 수정하는 경우 항상 이 작업을 하세요. 백업 파일과 함께 이 파일을 다른 곳에 저장하세요. (중요 문서와 마찬가지로 다른 컴퓨터에 저장해 두시면 좋습니다.) **이 파일이 없어지면 얼마나 오래 걸려야 다시 파일을 만들 수 있을지 생각해 보세요.**

> **중요:** **도구모음/작업 영역을 수정할 때 Rhino 창이 하나만 열려 있어야 합니다!**  도구모음 설치가 완료된 후에, 도구모음 레이아웃 대화상자의 파일 > 저장 을 사용하여 변경 내역을 저장하세요. 또는 Rhino를 종료하고 다시 엽니다. Rhino는 종료시 자동으로 도구모음 변경 내역을 저장합니다. Rhino를 여러 창에서 열어둔 경우, 마지막으로 닫는 Rhino가 이전에 저장된 내역을 덮어쓰므로 기존에 변경해둔 내역이 사라질 수 있습니다.
