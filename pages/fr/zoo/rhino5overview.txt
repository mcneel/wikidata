=====Vue d'ensemble=====
Le Zoo permet de partager des licences de Rhino, Flamingo, Penguin, Brazil et Bongo (Codes CD) avec les utilisateurs du même groupe de travail. Lorsque ces produits sont installés comme noeuds de groupes de travail au lieu d'être installés de façon autonome, le système fonctionne ainsi : 

  * Lorsqu'un noeud du groupe de travail est lancé, une licence est demandée au Zoo. 
  * Une licence libre est assignée au nœud. 
  * Lorsque un nœud est fermé, la licence est rajoutée à la liste des licences dans le Zoo. 

** Le Zoo est gratuit !**

//Note : il est possible d'utiliser Zoo 4.0 et Zoo 5 sur le même réseau et même de les installer sur le même serveur sans entraîner de conflit.//

=====Fonctions et caractéristiques=====
  * **Pas de coût supplémentaire** - Aucune version spéciale de Rhino, Flamingo, Penguin ou Bongo n'est nécessaire. 
  * **Aucun matériel spécial n'est nécessaire** - Le Zoo fonctionne sur tous les systèmes Windows. 
  * **Aucun logiciel spécial n'est nécessaire** - Le Zoo fonctionne sur tous les systèmes Windows équipés de Microsoft .Net Framework 4, y compris Windows XP et toutes les versions suivantes. 
  * **Configuration simple** - Installez le Zoo sur un ordinateur fonctionnant sous Windows et tapez les clefs de licence dans le zoo au lieu de les taper dans chaque système. 
  * **Retrait** - Les clés de licence peuvent être retirées afin que les portables puissent être déconnectés du réseau. Le code peut être réintroduit dans le zoo quand le portable est reconnecté au réseau. 
  * **Robuste** - Les nœuds de réseau continueront à fonctionner même si la connexion au réseau est interrompue ou si le serveur s'arrête.
  * **Compatible réseau routeur** - Le Zoo fonctionne aussi bien sur les réseaux à un seul segment que sur les réseaux où les systèmes sont séparés par des routeurs. 
  * **Un service** - Si vous redémarrez le système où est installé le Zoo, celui-ci démarrera automatiquement. 

=====Nouveautés du Zoo 5=====
  * **Nouvelle version.** Zoo 5 est une toute nouvelle version du Zoo. Entièrement réécrit, Zoo 5 apporte quelques nouvelles fonctions par rapport aux versions précédentes, telles que
  * **Prise en charge du protocole IP standard** - Les clients communiquent avec Zoo 5 en utilisant le protocole **HTTP (Port 80)**. Cette nouveauté permet aux administrateurs de système de rediriger plus facilement les requêtes du Zoo sur des réseaux complexes, voire même sur Internet.
  * **Compatible avec le pare-feu** - Pour que Zoo 5 fonctionne, le client doit consulter périodiquement le serveur mais il ne débute jamais de connexion vers le client. Les administrateurs n'ont plus besoin d'ouvrir des ports sur les ordinateurs clients.
  * **Limite de retrait de licences** - La version précédente de Zoo permettait de retirer des licences pour les ordinateurs portables. Mais il n'existait aucune limite en termes de durée. Zoo 5 permet à l'administrateur d'indiquer la durée du retrait des licences.
  * **Prise en charge des modules tiers** - La version précédente de Zoo ne traitait que les licences des produits de McNeel Zoo 5 permet aux développeurs de modules d'ajouter une prise en charge de leurs produits par le Zoo.
