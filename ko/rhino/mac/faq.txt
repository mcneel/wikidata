====== Mac Rhino 질문과 대답(FAQ) ======

--> Mac용 Rhino를 구입할 수 있습니까#

아니요. Mac용 Rhino는 현재 개발 중입니다. 아직 판매되고 있지 않습니다. [[http://www.rhino3d.com/download/wenatchee/5.0/wip|Mac용 Rhino를 다운로드]]하시면 개발 기간 동안 사용하실 수 있습니다.  여러분의 의견과 제안을 환영합니다.

<--

--> Mac Rhino에는 시간 제한이 있습니다. 시간이 만료된 후에는 어떻게 됩니까?#

시간 제한은 모든 분들께서 최신 버전을 테스트하고, 따라서 이미 수정된 문제를 반복해서 보고하시는 일을 방지합니다. 

각각의 Mac Rhino WIP 버전이 만료되기 전에 새로운 WIP 버전을 [[http://www.rhino3d.com/download/wenatchee/5.0/wip|다운로드]]하실 수 있습니다. Mac Rhino는 매번 시작할 때마다 WIP 업데이트를 확인하고 새 버전이 있으면 사용자에게 알려 드립니다. 새 WIP 버전이 업데이트되면 [[http://discourse.mcneel.com/ |Discourse 포럼]]의   [[http://discourse.mcneel.com/category/rhino-for-mac|Rhino for Mac]] 항목에 포스팅됩니다.

<-- 

-->Windows 버전과 데이터 파일이 호환됩니까?#

예. Rhino 5 파일은 동일합니다. Rhino 4 파일 또한 Windows Rhino 4용으로 내보낼 수 (export) 있습니다.

<-- 

-->Macintosh 버전에 2버튼식/ 3버튼식 마우스를 사용할 수 있습니까?#

예. 모든 Macintosh 컴퓨터는 멀티 버튼과 스크롤 휠이 있는 마우스를 지원합니다. 버튼이 1개인 Mac 마우스도 사용하실 수 있습니다. Control 키를 누른 상태에서 단추를 클릭하면, 마우스 오른쪽 단추 클릭처럼 실행됩니다.

<-- 

-->Macintosh 버전에서 Spaceball/SpaceNavigator/SpaceExplorer 를 사용할 수 있습니까?#

현재 SpaceNavigator만 지원됩니다.

<--
 


-->무엇이 안 됩니까? "Unknown or unimplemented command (알 수 없거나, 아직 구현되지 않은 명령)"란 메시지가 왜 표시됩니까?#

Windows용 Rhino 프로그램의 기능을 Macintosh 플랫폼에서도 가능한한 많이 지원하려고 계획하고 있으나, 아직 Macintosh WIP 버전에서 모든 기능들이 구현된 것은 아닙니다. 새 제품이 어떻게 개발되는지에 대한 안내는 Windows Rhino의 [[http://www.rhino3d.com/nr.htm|소프트웨어 개발 과정]]을 참조하시기 바랍니다. Macintosh 버전에도 동일한 개발 과정이 적용됩니다. 현재 [[http://www.rhino3d.com/nr.htm#phase1|1단계: 개발 중(WIP: work-in-progress)]] 입니다. 

이 [[rhino:mac:whatworks|웹페이지]]에서 현재 실행되는 기능과 실행되지 않는 기능에 대한 설명이 소개되어 있습니다.

<-- 

-->Macintosh 버전이 영어 이외의 언어로도 실행됩니까?#

예. 현재 한국어, 스페인어, 프랑스어, 독일어, 이탈리아어, 중국어, 일본어가 지원됩니다. 이 버전은 WIP (개발 중:Work In Progress) 이므로, 번역 작업이 필요한 부분이 있을 수 있습니다. 

Macintosh Rhino는 Macintosh에 기본 언어로 설정된 언어를 기본적으로 사용합니다. OS X 기본 언어와 다른 언어를 사용하려면 어플리케이션 메뉴에서 다음과 같이 설정하세요.
  - 메뉴에서 **Rhinoceros**를 클릭하고 **설정**을 클릭합니다.
  - **언어 환경설정** 패널에서 사용하시는 언어를 선택합니다.

<-- 
-->Rhino를 설치 제거하는 방법?#

다음 과정대로 실행하시면 컴퓨터에서 Mac용 Rhino가 완전히 제거됩니다:

  - Application 폴더에서 Rhino를 제거합니다. 
  - **/Library/Application Support/McNeel** 디렉터리를 제거합니다. 튜토리얼 모델과 사용자 가이드를 다운로드하셨다면 해당 파일이 이 디렉터리에 있습니다. 이 디렉터리가 없을 수도 있습니다.
  - 홈 디렉터리에서 **~/Library/Application Support/McNeel** 디렉터리를 제거합니다. 이 디렉터리는 Rhino에서 선택적으로 사용되며, 사용자의 컴퓨터에 없을 수도 있습니다. 사용하신 Toucan 재질, Python 스크립트, 다른 Mac Rhino 플러그인 등이 포함되어 있을 수 있습니다.
  - 홈 디렉터리에서 **~/Library/Preferences/com.mcneel.rhinoceros.plist** 파일을 삭제합니다. 이 파일에는 모든 Rhino 설정이 포함되어 있습니다.

<-- 
-->언제 Mac용 Rhino가 릴리스됩니까?#

아직 미정입니다. McNeel & Associates는 남다른 개발 방식을 채택하였습니다. 아직 미완성인 개발 초기 단계에 소프트웨어를 공개하여 여러분의 의견과 제안을 받아 이를 제품에 반영시킵니다. Mac Rhino는 [[http://www.rhino3d.com/nr.htm|Windows Rhino 개발]]과 동일한 과정으로 개발되고 있습니다.

<--
 
-->Windows용 Rhino를 구입하면 Mac용 Rhino도 무료로 받을 수 있나요? 양쪽 시스템에서 업데이트는 어떻게 합니까?#

업그레이드 정책에 대해서는 아직 아무것도 정해진 바가 없습니다.

==== 스크립트와 플러그인 ====

-->Macintosh 버전에서도 명령 스크립트를 실행할 수 있습니까?#

예. 명령 스크립트를 로드하고 실행하려면 Rhino 메뉴에서 **도구 > 명령 > 파일에서 읽기**를 선택합니다.
 
<--

-->Macintosh 버전에서 RhinoScript 프로그램을 실행할 수 있습니까?#

아니요. RhinoScript는 VisualBasic을 기본으로 하며, VisualBasic은 OS X에서 실행되지 않습니다. Mac과 Windows 버전에서 모두 스크립팅하는 데 Python을 사용하고 있으며, Python을 사용하시면 두 버전의 Rhino에서 사용 가능한 스크립트를 작성할 수 있을 것입니다. Windows와 OS X에서 Python 스크립팅 지원에 대한 안내는 [[http://discourse.mcneel.com/category/scripting | Rhino 스크립팅 지원 포럼]]을 참조하세요.

<--


-->Mac Rhino에서 서드 파티 (third-party) 플러그인이 실행됩니까?#

아마도 그럴 것입니다. 그러나, Windows 용 플러그인은 Mac에서 실행되지 않습니다. 플러그인을 Mac Rhino로 포트할지의 여부는 해당 플러그인 개발자가 결정하는 사항입니다.
<--
 
-->Macintosh 버전에서 C 플러그인이 지원됩니까?#

궁극적으로는 지원이 될 것입니다. 그러나 아직, C 플러그인에 대해 지원해 드릴 준비가 되지 않았습니다. 앞으로 지원할 계획을 갖고 있습니다. 여러분이 직접 Macintosh 개발 시스템을 사용하여 해당 플러그인을 다시 컴파일해야 합니다. 플러그인의 복잡한 정도에 따라 다르겠으나, Windows와 Macintosh에서 모두 실행되도록 컴파일하려면 추가적인 작업이 다소 필요할 수 있습니다. 

<--

-->Macintosh 버전에서 .NET 플러그인이 지원됩니까?#

궁극적으로는 지원이 될 것입니다. Rhino 플러그인 개발자들이 OS X와 Windows에서 모두 실행되는 플러그인을 작성할 수 있게 만드는 툴을 당사에서 개발 중입니다.

<-- 

-->OS X에 "The following disk images couldn't be opened (다음 디스크 이미지를 열 수 없습니다)" 라는 메시지가 표시됩니다. 어떻게 해야 합니까?#

아마도 다운로드한 .dmg 파일이 손상된 것으로 보입니다.   다운로드하신 .dmg 파일이 유효한지 확인하세요. 간혹, 다운로드하신 파일이 손상되어 있는 경우가 있습니다.

  - 디스크 유틸리티 프로그램 **/Applications/Utilities/Disk Utility** 을 시작합니다.
  - 메뉴에서 **Images > Verify...**를 선택하면 Open File 대화상자가 표시됩니다.
  - 다운로드한 dmg 파일을 선택합니다. 일반적으로 다운로드한 파일은 Downloads 폴더에 있으며 파일 이름은  rhinoceros_wip_YYYYMMDD.dmg (YYYY 연도, MM 월, DD 일) 로 구성되어 있습니다.
  - 다운로드한 파일이 유효하지 않다면  [[http://www.rhino3d.com/download/wenatchee/5.0/wip|다운로드]] 웹페이지를 방문하시고 새로운 다운로드 링크를 요청하시기 바랍니다.
  - 이러한 현상이 다시 발생한다면, 귀하께서 사용하시는 인터넷 연결이 불안정하기 때문이므로, 다른 인터넷 연결에서 다시 파일을 다운로드하셔야 합니다.  다운로드 파일은 100 MB를 넘습니다.

<-- 

-->Rhino 최신 버전을 설치하였으나 만료됐다는 메시지가 표시됩니다. 어떻게 해야 합니까?#

귀하의 컴퓨터에는 2개 이상의 Rhino가 설치되어 있으며, 만료된 버전부터 시작하므로 해당 메시지가 표시됩니다. **모든** Rhino를 제거하셔야 합니다. 

아무 3DM 파일을 두 번 클릭합니다. 해당 파일이 Rhino에서 열리면 어딘가 Rhino가 설치되어 있기 때문입니다. 
  - Dock에서 Rhino 아이콘을 두 번 클릭하고 메뉴에서 **옵션 > 파인더에서 표시(Show in Finder)**를 선택하여 위치를 찾습니다.
  - 찾을 Rhino를 Trash로 드래그합니다. 
  - 더 이상 Rhino가 없을 때, 3DM 파일을 두 번 클릭하면 파일이 TextEdit 에서 열립니다. 이제 모든 Rhino가 제거되었습니다.
  - 방금 다운로드하신 Rhino 버전을 다시 설치하세요.

<--
 
-->Rhino를 Applications 폴더로 드래그하면 BGHUDAppKit 가 실행 중이어서 설치할 수 없다는 메시지가 표시됩니다. 무슨 뜻이죠?#

이것은 10.7 파인더에 있는 오류입니다. 복사본을 시작하기 전에 어플리케이션을 덮어 쓸 수 있는지 올바르게 확인하지 않습니다. 이 오류 메시지가 표시되거나, Info.plist에 대한 오류 메시지가 표시될 수 있습니다.

안타깝게도 이 메시지가 표시되는 것은 파인더가 기존 Rhino 어플리케이션의 일부를 벌써 삭제하고 새 버전을 설치하지 않았음을 뜻합니다.  즉, 현재 Rhinoceros가 손상되어 다시 실행하려고 하면 크래시가 발생하게 됩니다.

드래그 앤 드롭을 두 세 차례 실행하면 결국엔 실행됩니다. 새 버전을 설치하기 전에 Applications 폴더에 있는 Rhinoceros를 Trash 로 드래그할 수도 있습니다. 이제 파인더가 기존 버전을 덮어 쓰는 데 문제가 없습니다.
 
10.8 파인더는 이 작업을 올바르게 실행합니다. 10.8로 업그레이드하는 것도 또 다른 해결책입니다.

<-- 

-->"Program Failure"라는 이상한 메시지가 표시되고 난 후, Mac Rhino가 더 이상 실행되지 않습니다.#

Mac Rhino에서 예상하지 못 한 상황에 부딪히면 일반적인 "Program Failure" 메시지가 표시됩니다. 이 오류가 발생한 후에는 Mac Rhino가 내부적으로 종료되고 이 시점 이후 명령 또는 메뉴를 실행하거나 선택할 수 없게 됩니다. 파일을 저장하고 Mac Rhino를 끌 수 있지만, 다른 작업은 불가능합니다. 이 문제가 언제 발생했는지 저희에게 알려 주셔야 합니다. 
Rhino에서 작업하실 때 다음 메시지를 보실 수 있습니다:

몇 가지 상황에서 일반적인 메시지가 Rhino에 표시됩니다. Rhino에 아직 모든 기능이 구현된 것은 아닙니다. 아직 구현되지 않은 기능을 사용하려고 하시면 이 메시지가 표시됩니다. Windows용 Rhino에서만 사용되어야 하는 프로그램 코드도 있습니다. 이 코드를 실행시키는 어떤 작업을 하시게 되면 "Rhino Error" 메시지가 표시됩니다. 마지막으로, 초기 코드에는 많은 디버깅 테스트가 있습니다. 발생해서는 안 되는 상황을 내부 테스트에서 발견하게 되면 이 메시지가 표시됩니다.

{{:rhino:mac:notimpl.png?nolink|}}

이들 메시지를 보시게 되면 해당 메시지에서 파일 이름, 줄 번호, 버전 정보를 기록해 두세요. 이 정보와, 이 메시지가 표시되었을 때 어떤 상황이었는지에 대한 설명을 적어 <mactech@mcneel.com>에게 이메일로 보내주세요. 해당 문제가 저희쪽에서도 발생되는지 확인하고 해결책을 찾아 다음 번 릴리스에 반영시키도록 하겠습니다.

이 메시지를 받으신 후에는 Rhino에서 몇 가지 명령이 제한될 수 있습니다. 파일을 저장하고 Rhino를 끌 수는 있으나 그 외 많은 작업이 안 될 것입니다.

<-- 


--> 오른쪽 클릭으로 마지막 실행된 명령을 다시 실행하고 싶습니다 #

상황에 맞는 메뉴(context menu)를 사용 안 함으로 설정하시면 이 작업을 하실 수 있습니다.

  - 메뉴에서 **Rhinoceros**를 클릭하고 **설정**을 클릭합니다.
  - **마우스** 패널을 선택합니다.
  - **마우스 오른쪽 단추** 항목에서 **Enable Context Menus** 확인란을 선택 해제합니다.

<--

--> 도구 단추에서 두 번째 명령을 오른쪽 클릭으로 실행하고 싶습니다#

일부 툴 팔레트 단추에는 해당 단추에 2개의 명령이 있습니다. 명령은 도구설명으로 표시됩니다.
두 번째 명령을 실행하는 기본 방법은 해당 도구 단추를 Option 키와 함께 클릭하는 것입니다.

  - 메뉴에서 **Rhinoceros**를 클릭하고 **설정**을 클릭합니다.
  - **툴 팔레트** 패널을 선택합니다.
  - **Execute alternate command when** 설정을 지정합니다. 또한, 도구모음과 툴 팔레트는 오른쪽 클릭을 같은 방식으로 처리합니다.

**안내:** 오른쪽 클릭을 두 번째 명령이 실행되도록 정의했다면 오른쪽 클릭으로 문서 도구모음을 사용자 지정할 수 없습니다.  View/Customize Toolbar 메뉴 선택을 대신 사용해야 합니다.

Mac UI 표준에서 오른쪽 클릭(또는 control-클릭)은 //항상// 컨텍스트 메뉴를 엽니다.  이 툴 팔레트 패널에서 원하는 대로 설정을 변경하실 수 있습니다.
