======어떻게 Rhino 5가 Zoo 서버를 찾습니까?======
> **제품:** //[[ko:zoo:zoo5|Zoo 5]]//
> **요약:** //Rhino 5가 Zoo 5 라이선스 관리자 서버를 찾는 방법에 대한 개요입니다.//

=====개요=====
동일한 서브넷에 있으면 Zoo 4.0을 자동으로 찾을 수 있는 Rhino 4.0 과는 달리, Rhino 5는 Zoo 5 서버의 이름을 알아야 합니다.

Rhino 5가 Zoo 5 서버에서 라이선스를 필요로 할 때, 다음 위치를 순서대로 찾아 Zoo 5 서버의 이름을 확인합니다:
  - Windows 레지스트리의 Current User 하이브에서 Zoo 서버의 호스트 이름 또는 IP 주소를 찾습니다.
  - Windows 레지스트리의 Local Machine 하이브에서 Zoo 서버의 호스트 이름 또는 IP 주소를 찾습니다.
  - 기본 Zoo 서버 이름을 Domain Name System (DNS)에 문의합니다.
 

=====추가 정보=====

====레지스트리 조회 - Current User====

Windows 레지스트리 HKEY_CURRENT_USER 에서 Zoo 서버의 호스트 이름 또는 IP 주소를 찾을 때, Rhino 5 32 비트와 Rhino 5 64 비트 모두 다음 위치를 찾습니다:

<code>
Hive:  HKEY_CURRENT_USER
Key:   Software\McNeel\Rhinoceros\5.0\License Manager
Name:  Server
Type:  REG_SZ
Value: <host name or IP address>
</code>

====레지스트리 조회 - Local Machine====
Windows 32 비트 버전을 사용 중이라면, Windows 레지스트리 HKEY_LOCAL_MACHINE 에서 Zoo 서버의 호스트 이름 또는 IP 주소를 찾을 때  Rhino 5 32 비트가 다음 위치를 찾습니다:

<code>
Hive:  HKEY_LOCAL_MACHINE
Key:   Software\McNeel\Rhinoceros\5.0\License Manager
Name:  Server
Type:  REG_SZ
Value: <host name or IP address>
</code>

Windows 64 비트 버전을 사용 중이라면, Windows 레지스트리 HKEY_LOCAL_MACHINE 에서 Zoo 서버의 호스트 이름 또는 IP 주소를 찾을 때  Rhino 5 32 비트와 Rhino 5 64 비트 __모두__ 다음 위치를 찾습니다:

<code>
Hive:  HKEY_LOCAL_MACHINE
Key:   SOFTWARE\Wow6432Node\McNeel\Rhinoceros\5.0\License Manager
Name:  Server
Type:  REG_SZ
Value: <host name or IP address>
</code>

====DNS 사용하기====
Windows 레지스트리 조회를 통해 Rhino 5가 Zoo 5 서버의 이름을 확인하는 데 실패하면 다음의 호스트 이름으로 기본 Zoo 서버를 DNS 서버에 문의합니다:

<code>
__mcneel.__zoo5
</code>

예를 들어, 회사의 도메인이 "mcneel.com"이면, Zoo 서버에게 위치를 알려주기 위해 DNS 앨리어스인 <nowiki>"__mcneel.__zoo5.mcneel.com"</nowiki>을 추가할 수 있습니다.
