====== Problèmes d'installation ======

> **Sommaire:** //Quelques problèmes rencontrés lors de l'installation et les instructions pour les réparer//

=====Problème avec les configurations de barre d'outils qui ne sont pas ancrées correctement.=====

Fermer Rhino

Supprimer la clé suivante dans le registre

//HKEY_CURRENT_USER\Software\McNeel\Rhinoceros\3.0\Scheme : Default\Control bar manager//

Redémarrer Rhino

=====Problèmes de lancement de Rhino après avoir installé Rhino 3.0 SR4 ou la bêta de la 4.0.=====
Ce problème peut aussi avoir un rapport avec l'installation/le lancement de Windows XP SP2

----
Symptômes - Rhino ne s'ouvre pas après l'installation d'un des éléments précédents.

Solution à essayer :

Copier les fichiers //msvcp71.dll// et //msvcr71.dll//

de

//C:\Program Files\Rhinoceros 4.0\System//

vers

//C:\Program Files\Common Files\McNeel Shared\License Manager//

Ensuite, Rhino 3.0 et 4.0 devraient fonctionner correctement.

Écrivez un message dans le forum de Rhino si cette solution ne corrige pas le problème.


=====Impossible d'installer ou de désinstaller Rhino 4.0 Bêta=====

PROBLÈME :

Quand j'essaie d'installer Rhino 4.0, j'obtiens un message de l'installateur de Windows indiquant que le produit doit être désinstallé en utilisant Ajout/Suppression de programmes et que l'installation ne continuera pas.

Mais quand j'essaie de désinstaller avec Ajout/Suppression de programmes, ça ne marche pas non plus.

SOLUTION :

Cette solution désactive les mesures de sécurité de l'installateur de Windows, attention donc à ne l'utiliser que si vous comptez réinstaller le produit immédiatement après.

Utiliser REGEDIT.EXE pour supprimer la clé de registre (faire attention !) :

HKEY_CLASSES_ROOT\Installer\Products\DFFBC2C5B3CF9AA499B3ECB2D82AB578


=====L'index de l'aide de Rhino 3.0 est vide=====

PROBLÈME :

L'index de l'aide de Rhino ou de la fenêtre de l'aide des outils extra est vide pour les utilisateurs ayant des droits restreints dans Windows 2000.

SOLUTION :

Il s'agit d'un problème de permissions qui s'ajoute à un bogue de Windows 2000.

Voici comment le réparer sans changer les permissions dans le dossier des fichiers de programme.

Demander à un administrateur de supprimer le fichier Rhino.GID dans le dossier  ...Program Files\Rhino\English\Help et pendant que vous y êtes supprimer aussi RhinoBonusTools.GID

Fermer la session de l'administrateur (ne pas lancer l'aide avant sinon le fichier GID sera recréé pour l'administrateur).

Ouvrir la session de l'utilisateur avec des droits restreints. L'index devrait maintenant être affiché (et un fichier GID est créé quelque part dans la zone de fichiers personnels de l'utilisateur.)

Cette opération n'est pas nécessaire sur Windows XP, Microsoft a corrigé le bogue.

Une autre façon de corriger ce problème dans Windows 2000 est d'ajouter la permission d'écrire pour tous les utilisateurs dans le dossier ...Rhino\English\Help
