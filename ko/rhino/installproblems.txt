=====Rhino 5 설치 관리자 문제 보고=====
Rhino 5 설치 과정이 실패하면 로그 파일이 표시됩니다. 이 로그 파일은 저희 기술 지원 담당자가 사용자의 설치 문제를 해결하는 데 큰 도움이 됩니다. 
  - 오류 메시지가 표시되면 대화상자에 링크가 있습니다. 
  - 링크를 클릭하여 로그가 올리면 txt 파일로 저장합니다.
  - 텍스트 파일을 이메일에 첨부하여 [[tech@mcneel.com]]로 보냅니다.
[[http://wiki.mcneel.com/ko/rhino/installproblems|맨 위로 이동.]]
=====Rhino 5 설치 관리자 오류 1603 등=====
로그의 **오류 코드 1603** 은 특정한 크래시 오류가 아니라, 설치 과정 중에 발생하는 일반적인 오류입니다.  1603 오류를 일으킬 수 있는 조건들은 다음과 같습니다:
  * 대상 컴퓨터에 짧은 파일 이름 생성이 허용되어 있지 않습니다.
  * Install Script 사용자 지정 작업이 잘못 프로토타입화되었습니다.
  * 파일이 잠겨 있어 덮어쓸 수 없습니다.
  * Microsoft Windows Installer Service가 올바르게 설치되어 있지 않습니다.
  * Windows Temp 폴더가 가득 차있습니다.
  * 설치한 후에 설정이 손상되어, 설치 제거할 때 이 오류로 인해 설치 제거가 안 됩니다.
Rhino 설치 관리자에 오류 코드 없이 또는 1603 오류와 함께 크래시가 발생하면 다음 과정을 실행하여 더 자세한 정보를 알아보시기 바랍니다.

**//1 단계//**
  - 위에 안내되어 있는 Rhino 5 설치 관리자 오류 아래의 과정을 실행하고 저희에게 설치 로그 파일을 보내 주세요. 
  - 기존 설치 관리자를 닫습니다.
  - Windows 탐색기를 열어 Rhino 설치 관리자 DVD의 내용을 찾습니다.
  - DVD의 Installer 폴더를 엽니다.
  - Prerequisites 폴더를 엽니다.
  - 해당 폴더 안에 있는 [[http://www.microsoft.com/en-us/download/details.aspx?id=17718|dotNetFx40_Full_x86_x64.exe]] 을 실행합니다.
  - dotNet 설치 관리자에서 오류가 표시되면 오류 내용을 그래도 기록하고 2단계로 넘어갑니다.
  - dotNet 설치 관리자가 성공적으로 완료되면 3단계로 넘어갑니다.

**//2단계//**
  - [[http://windows.microsoft.com/en-US/windows/help/windows-update|Windows 업데이트 서비스]]를 실행합니다.
  - 설치해야 할 중요 업데이트가 있으면 **지금 업데이트**를 클릭하여 업데이트를 설치합니다. 이 과정은 오랜 시간이 걸릴 수 있습니다.
  - Windows 업데이트 설치 관리자를 실행하려고 할 때 오류가 발생하면 [[http://support.microsoft.com/kb/906602/ko|공용 Windows Update, Microsoft Update 및 Windows Server 업데이트 설치 문제를 해결하는 방법]]을 참조하세요.
  - Windows 업데이트 문제를 해결하고 [[http://windows.microsoft.com/en-US/windows/help/windows-update|Windows 업데이트 서비스]] 실행을 완료했다면 1단계로 되돌아갑니다.

**//3단계//**
  - Windows 탐색기에서 Rhino 설치 관리자 DVD로 갑니다.
  - DVD 루트의 Setup.exe 을 실행합니다.
  - 설치가 완료되어야 합니다.
  - 설치 관리자에서 계속해서 크래시가 발생하면 이메일에 로그 파일을 첨부하여  tech@mcneel.com 앞으로 보내 주세요.

=====키를 열 수 없습니다: UNKNOWN\Components (오류 1402 또는 1406)=====
** //질문://** // "키를 열 수 없습니다: UNKNOWN\Components\(많은 글자와 숫자). 오류 1402. 해당 키에 대한 액세스 권한이 충분한지 확인하거나 고객 지원 담당자에게 문의하십시오."//

**//원인://** HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Installer\UserData\S-1-5-<numbers>\Components key 의 하나 이상의 키에 잘못된 권한이 있습니다.

**//해결책 1://**
Windows 시작 메뉴의 검색 상자에 CMD를 입력합니다. cmd.exe를 오른쪽 클릭하고 "관리자 권한으로 실행"을 선택합니다. 다음에 적힌 부분을 복사하야 cmd 창에 붙여넣은 후 Enter 키를 누릅니다. 이제 Rhino를 설치하실 수 있습니다.

<code>
secedit /configure /cfg %windir%\inf\defltbase.inf /db defltbase.sdb /verbose
</code>

**//해결책 2://**
//경고: 다음 과정은 Windows 레지스트리 수정을 요합니다. 레지스트리를 잘못 수정하면 컴퓨터에 손상이 가고, 그로 인해 아예 컴퓨터가 시작하지 않을 수 있습니다. 다음 과정을 실행하는 것이 부담스럽다면 McNeel 기술 지원에 문의하세요.//
  - Microsoft TechNet에서 [[http://technet.microsoft.com/en-us/sysinternals/bb897553|psexec]] 을 다운로드하고 바탕 화면에 압축 해제하세요.
  - Windows 명령 셸을 엽니다.
  - 디렉터리를 바탕화면으로 변경합니다.
  - 명령 셸에 다음을 입력합니다: **psexec -i -d -s c:\windows\regedit.exe** 이것은 Windows 에서 시스템 계정 권한으로 레지스트리 편집기를 실행하게 합니다. 시스템 계정에는 관리자 계정보다 훨씬 강력한 권한이 있습니다.
  - HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Installer\UserData\ 위치를 찾습니다.
  - **UserData** 폴더를 오른쪽 클릭하여 **사용 권한...** 을 클릭합니다.
  - UserData의 사용 권한 대화상자에서 **고급**을 클릭합니다.
  - "관리자"가 해당 폴더의 "모든 권한"을 갖고, "사용자"는 "읽기" 권한을 갖도록 설정합니다. 다른 사용자 권한도 있으나, 관리자와 사용자가 가장 중요합니다.
  - **모든 하위 개체 사용 권한을 이 개체의 상속 가능한 사용 권한으로 바꾸기** 를 선택합니다.
  - **확인**을 두 번 클릭하여 적용합니다.
  - 다시 Rhino를 설치합니다.

=====서비스 릴리스란?=====
서비스 릴리스는 소프트웨어의 기존 설치 관리자의 최신 오류가 수정되고 업데이트된 무료 패치 릴리스입니다. 이 설치 관리자는 다운로드하실 수 있으며, 기존에 제품이 설치되어 있어야만 서비스 릴리스를 설치하실 수 있습니다.  서비스 릴리스는 SR3 또는 SR4b와 같은 SR 번호와 파일 이름에 코드되어 있는 날짜 (20050729는 2005 년 7월 29일을 의미)로 확인하실 수 있습니다.
\\
[[http://wiki.mcneel.com/ko/rhino/installproblems|맨 위로 이동.]]
=====설치된 버전이 어느 버전인지 어떻게 확인하나요?=====
소프트웨어를 설치하시면, 서비스 릴리스의 번호와 날짜를 볼 수 있습니다. 이 번호와 날짜는 Rhino 를 시작하면 "스플래시 화면"에 표시되며, Rhino > 도움말 > [[rhino:rhinoceros|Rhinoceros]] 정보 메뉴에서도 확인하실 수 있습니다.
\\
Flamingo에서는 FlamingoAbout 명령을 실행하세요. Bongo에서는 BongoAbout 명령을 실행하세요.
\\
[[http://wiki.mcneel.com/ko/rhino/installproblems|맨 위로 이동.]]
=====서비스 릴리스는 어디에 있습니까?=====
업데이트 확인 서비스를 사용하는 빠른 방법은 Rhino 의 도움말 메뉴에서 업데이트 확인 항목을 클릭하는 것입니다. 인터넷에 연결된 상태라면, 사용하고 계신 소프트웨어보다 최신 버전이 있는 경우 McNeel 웹 서버에서 올바른 서비스 릴리스를 다운로드하실 수 있도록 안내해 드립니다. Flamingo 에서는 Raytrace 또는 Photometric 메뉴에서 Check for Updates 를 선택하세요. Bongo 에서는 Bongo 메뉴에서 Check for Updates 를 선택하시면 됩니다.
\\
Rhino 5의 **도구, 옵션, 업데이트와 통계**에서 업데이트 간격을 "서비스 릴리스" 또는 (용감하시다면) "서비스 릴리스 후보"로 설정합니다. Rhino를 종료하면 최신 빌드의 설치를 안내하는 메시지가 표시됩니다. 지금이 아니라 차후에 설치하시려면 취소를 클릭하시면 됩니다.  
\\
[[http://wiki.mcneel.com/ko/rhino/installproblems|맨 위로 이동.]]
====="프로그램 추가 또는 제거"로 이전 버전을 문제 없이 제거할 수 있습니까?=====
Windows의 제어판에서 "프로그램 추가 또는 제거" 항목은 모든 서비스 릴리스를 각각 250 MB 디스크 공간을 사용하는, 설치된 프로그램으로 보고합니다. 이것은 잘못 이해하기 쉽습니다. 서비스 릴리스를 설치하면 대부분 또는 모든 파일이 기존 설치 파일을 덮어쓰지만 Windows에서는 기존 설치된 파일에 새로운 파일이 추가된 것처럼 합계를 내어 보고하게 됩니다. 오래된 서비스 릴리스를 제거하기 위해 "프로그램 추가 또는 제거"를 **사용하지 마십시오**."
\\
[[http://wiki.mcneel.com/ko/rhino/installproblems|맨 위로 이동.]]

=====오류 1327 - 유효하지 않은 드라이브:=====
**//질문://** //설치할 때 "오류 1327. 유효하지 않은 드라이브 (E:, F:, etc)" 오류가 표시됩니다//

**//원인://** //사용자 계정에 하나 이상의 존재하지 않는 드라이브에서 "내 문서"와 같은 특별한 폴더를 사용해야 하는 설정이 있기 때문입니다.//

**//해결책://**
  - "시작" 메뉴에서 "실행"을 클릭합니다.
  - RegEdit 를 입력하고 "확인"을 클릭합니다.
  - HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Shell Folders 위치를 찾습니다.
  - 설치 관리자의 오류에 언급되어 있는 드라이버 글자가 포함되어 있는 디렉터리를 찾습니다. 이것을 올바른 드라이브와 폴더로 지정합니다.
  - HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders 위치를 찾습니다.
  - 오류에 언급된 드라이버 글자가 포함되어 있는 디렉터리를 찾습니다. 

**안내:** 이 방법 또는 작업에는 레지스트리를 수정하는 방법이 포함되어 있습니다. 레지스트리를 잘못 수정하면 심각한 문제가 발생할 수 있습니다. 따라서 주의하여 이 과정을 실행하시기 바랍니다. 레지스트리를 수정하시기 전에 레지스트리를 백업하시면, 문제 발생 시 원래 레지스트리를 복구하실 수 있습니다. 레지스트리의 백업과 복구에 대한 자세한 정보는 Microsoft Knowledge Base의 다음 글을 클릭하셔서 참조하시기 바랍니다:
[[http://support.microsoft.com/kb/322756|How to back up and restore the registry in Windows ]]

**해결책:**

  - .NET Framework Cleanup 도구를 실행합니다.
  - 정리할 제품을 선택할 때 **.NET Framework - All versions**을 선택합니다.\\ {{http://wiki.mcneel.com/_media/rhino/rhino5/k01521.png?w=405}}
  - **Cleanup Now** 단추를 클릭합니다.
  - 컴퓨터를 다시 시작합니다.
  - Rhino 5 DVD에서 Rhino 5를 설치합니다.\\ <color #999999>Rhino 5 DVD 설치 관리자가 .NET Framework 4 를 먼저 설치한 후 Rhino 5를 설치합니다.\\ </color><color black>Rhino 5 평가판</color><color #999999>을 설치하려고 한다면, Rhino 5를 설치하기 전에 [[http://www.microsoft.com/en-us/download/details.aspx?id=17718|.NET Framework 4 전체 버전을 다운로드하여]] 다시 설치하셔야 합니다. </color>

===== Windows 7/8 이 지원되는 시스템이 아닙니다? =====
**문제:**\\ Windows 7/8 을 사용 중인데, Rhino 5 DVD 설치 관리자에서 지원되지 않는 시스템이라고 나옵니다. 

**해결책:**\\ 
  - exe 설치 파일을 오른쪽 클릭하고 **속성**을 선택합니다.
  - **호환성** 탭으로 갑니다.\\ <color #999999>호환성 모드를 "Windows XP (서비스 팩 2)"를 선택할 수 있으나 변경할 수 없습니다.</color>
  - 아래쪽의 **모든 사용자에 대한 설정 변경** 단추를 클릭합니다.
  - **호환 모드로 이 프로그램 실행**  확인란의 선택을 해제합니다. 
  - **확인**을 두 번 클릭하여 대화상자를 닫습니다.
  - 다시 Exe 설치 관리자에서 오른쪽 클릭하고 **관리자 권한으로 실행**을 선택하여 설치합니다.
