====== 常見回覆 -- Rhino ======

==我如何寄送一個錯誤回報？ ==
  * 您能重複這個錯誤嗎？若可以，程式開發者將會需要您詳細說明來重現這個錯誤。可重複發生的問題是修正並發現軟體錯誤最佳的方式。
  * 當錯誤回報系統畫面出現，若您尚有足夠的記憶，請填入您的email並且描述Rhino當機時您正在做那些動作，若能提供我們一個在當機之前的簡單的指令清單，對於我們查錯將會相當有幫助。
  * 若可能的話，請勾選寄送您的3dm檔案。
  * 若您提供您的email及描述， **您的回報將有專人分析**。若您不填寫您的email及描述，我們將使用自動化流程分析您的錯誤回報。

==安全模式(Safe Mode)==
  *在Rhinoceros程式集有包含一個安全模式的捷徑。 安全模式是疑難排除時使用的工具，開啟時不載入OpenGL及外掛程式。於Rhino 5同時也可不載入工具列及RDK。一般而言，我都會載入它們。安全模式是疑難排除時使用的工具，並不建議當作工作使用。

----
==停用V5外掛程式==
  -請到 工具->選項->外掛程式，並且取消勾選已啟用的外掛程式來停用外掛程式。
  -重新開啟Rhino，並且確認是否有當機情形，再重新啟用其中一個外掛程式，再重新開啟一次Rhino，看看是否會當機，若無，這代表啟用的這個外掛程式並沒有問題。請重複這個步驟逐一啟用外掛程式，直到找到有問題的外掛程式。
----

==無法開啟Rhino V5 ==
  *這個問題可能是顯示卡驅動程式或外掛程式有問題。可依照以下方式找出問題的來源，開啟Rhino 安全模式(請到開始->Rhino 5->安全模式)，並到工具->選項->視圖->OpenGL，取消勾選使用硬體加速模式。重新開啟一般的Rhino，若可以正常開機，這代表問題可能是顯示卡驅動程式，請更新電腦的顯卡驅動。更新後若仍然當機，那麼這代表問題應該是外掛程式所造成。

  *若是這種情況，重新啟動安全模式，重新將之前取消的使用硬體加速模式啟用。然後到工具->選項->外掛程式，並且將所有的非Rhino內建外掛程式取消啟用，並勾選下方的”詢問是否載入停用的外掛程式”，並且當成是詢問您是否載入，請不要載入所有的程式。重新開啟一般的Rhino，這次請載入第一個外掛程式，其餘的請不要載入，若載入第一個外掛並且當機，代表這個外掛出現問題。若沒有當機，代表這個外掛沒有問題。請重新再開啟一般的Rhino，繼續逐一測試其他的外掛程式，直到找到有問題的那一個為止。

==我可以將我購買的Rhino安裝在我的兩台電腦上(桌機與筆電)嗎？ ==
  * 可以的。授權書上寫明可以讓您將Rhino安裝在所有您可以直接控制的電腦上，唯同一時間您只能在一台電腦上使用Rhino。您不能將您的Rhino”借”給您的朋友使用。

==當開啟Rhino時，有無法讀取部分資料的錯誤訊息==
  * 系統作業分為兩個方式如下：
    - 自動儲存 – 若您有啟動Rhino自動儲存功能，暫存檔案會存在您在選項裡指定的資料匣路徑，並建立一個紀錄。當Rhino正常關閉，Rhino會將自動站存檔刪除並清除紀錄。若Rhino當機，這個紀錄將不會被清除，每次Rhino開啟時暫存檔都會存在。在這個時候，Rhino會出現不正常關閉的警示，並且提供您可以另存暫存檔的機會，若您回答不存檔，這個檔案將自動被刪除。
    - 備份 – 從Rhino 4.0 SR7開始，我們增加了一個自動備份的功能，若您沒有取消這個功能，每當您打開一個既有檔案，每當您開始編輯動作程式將在同一個資料匣自動儲存一個相同檔案名稱，副檔名為".bak"的備份檔。
  * Rhino 5 則產生名為*.3dmbak 檔案格式，並且在開啟舊檔可以直接開啟。
  * 在Rhino 4，您可以使用檔案總管，並且記得開啟顯示副檔名的選項。您可以使用更改檔案名稱的方式將 [檔名].3dm.bak 改成 [新檔名].3dm。然後就可以直接用Rhino開啟這個備份檔。

==當我安裝Rhino時，勾選了使用Zoo的選項，但我實際上並沒有使用Zoo ==
  * 若您使用Windows Vista 或 Windows 7，請刪除下列路徑的所有檔案：C:\ProgramData\McNeel\Rhinoceros\5.0\License Manager\Licenses 
  * 若您使用Windows XP ，請刪除下列路徑的所有檔案：C:\Documents and Settings\All Users\Application Data\McNeel\Rhinoceros\5.0\License Manager\Licenses
  - 重新開啟Rhino
  - 輸入您的序號。

備註：在某些系統，路徑中的"\All Users\"資料匣，可能是您的"\Domain\"登入名稱。
