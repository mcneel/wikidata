======Determinare il nome del server dello Zoo======
> **Prodotto:** //[[it:zoo:home|Zoo]]//
> **Sommario:** //Indica come determinare il nome dell'host del sistema su cui è in esecuzione lo Zoo.//

=====Domanda=====
Di recente ho acquistato due copie di Rhino 5 da condividere tra cinque sistemi. Ho quindi installato lo Zoo, a cui ho aggiunto le due licenze di Rhino.

Dopo aver installato Rhino, ho cercato di avviarlo. Nella finestra di dialogo apparsa, dopo aver cliccato sull'opzione **Rileva automaticamente server Zoo** e premuto il pulsante **Verifica connettività**...

{{:zoo:zoo_host0.png}}

...Rhino mi dice che è impossibile rilevare il server Zoo. Cosa devo fare?
=====Risposta=====
Se Rhino non è in grado di rilevare automaticamente il vostro server Zoo, perché non avete aggiunto una voce per il [[http://wiki.mcneel.com/zoo/findzooserver|server Zoo "di default"]] nel Server DNS della vostra azienda, basta che specifichiate manualmente il nome dell'host o l'Indirizzo IP del sistema su cui è in esecuzione lo Zoo.

=====Ulteriori informazioni=====
Se non sapete il nome dell'host del sistema su cui è in esecuzione lo Zoo, eccovi un paio di metodi per scoprirlo.

====1. Eseguite il comando Hostname====
Nel sistema su cui è in esecuzione il server Zoo, visualizzate una finestra del prompt dei comandi. Quindi, eseguite il comando **Hostname**.

Per esempio:

{{:zoo:zoo_host1.png}}

In questo esempio, il nome dell'host è "zoo".

====2. Eseguite ZooAdmin=====
Nel server Zoo, eseguite l'utilità di amministrazione dello Zoo o **ZooAdmin.exe**. La barra del titolo della finestra di dialogo dell'amministratore dello Zoo mostrerà il nome dell'host, assieme al nome di dominio, del sistema su cui è in esecuzione lo Zoo.

Per esempio:

{{:zoo:zoo_host2.png}}

In questo esempio, il nome completo dell'host è "zoo.mcneel.com".
