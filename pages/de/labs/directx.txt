====== DirectX 9 Rhino Labs Plug-in ======
> **Zusammenfassung:** //Mit diesem Plug-in können Rhino-Ansichtsfenster DirectX (vor allem Version 9 von DirectX) anstatt [[rhino:opengl|OpenGL]] verwenden. Ziel ist es, Rhino 4.0 Benutzern, die OpenGL-Probleme haben, eine Alternative anzubieten.//

=====Hintergrund=====

Der Zweck für das Hinzufügen von DirectX Support ist es, Benutzern zu helfen, die **ernste** Anzeigeprobleme mit [[rhino:opengl|OpenGL]] haben.

**Wenn Sie //KEINE// Probleme mit [[rhino:opengl|OpenGL]] haben, wird empfohlen, dass Sie [[rhino:opengl|OpenGL]] weiterhin für die Rhino-Anzeige verwenden.**

Es werden zwei Technologien, **OpenGL** und **DirectX**, verwendet, um Zugriff auf die Grafikkarte eines Computers zu haben und schnelle 3D-Zeichnung auf einem Ansichtsfenster auszuführen (auch bekannt als Hardware-beschleunigtes Zeichnen).

Rhino 4.0 verwendet standardmäßig **OpenGL**. Das funktioniert auf den meisten Systemen, aber einige Treiber-/Grafikkartenkombinationen könnten mit [[rhino:opengl|OpenGL]] schlecht oder gar nicht funktionieren.

Eine schlechte [[rhino:opengl|OpenGL]] Leistung unter **Windows Vista** ist ebenfalls bekannt, weil die meisten Grafikkartenhersteller noch keine OpenGL-Treiber für Vista veröffentlicht haben. Das könnte sich mit der Zeit ändern, sobald die Hersteller ihre Grafikkartentreiber aktualisieren, aber da Vista robusten DirectX-Support benötigt, erwarten wir, dass DirectX oft die beste Option für Vista sein wird.

DirectX ist eine Technologie, die [[rhino:opengl|OpenGL]] ähnlich ist. Das bedeutet, sie erlaubt einem Programm, Hardware-beschleunigte Grafiken zu verwenden.

Rhino wird den [[rhino:opengl|OpenGL]] Support **NICHT** einstellen.

=====Downloads=====

Am Ende wird das alles in einen einzigen Installer integriert, aber im Moment müssen Sie 3 Komponenten installieren, um das DirectX Plug-in auf Ihrem Computer zum Laufen zu bringen.

====Ein Mal:====
  * [[http://en.wiki.mcneel.com/content/upload/files/vcredist_x86.exe|Downloaden und installieren]] Sie die neusten Microsoft Runtime Bibliotheken.
  * [[http://www.microsoft.com/windows/directx/default.mspx|Downloaden und installieren]] Sie die neusten DirectX Endbenutzer Runtime Bibliotheken von Microsoft
    * Auch wenn Ihr Computer sagt, dass DirectX9c vorhanden ist, ist es empfehlenswert, diese Software runterzuladen und zu installieren.
====Bei jeder neuen Betaversion====
  * [[http://download.mcneel.com/download.asp?id=directx9|Downloaden und installieren]] Sie das DirectX 9.0 Anzeige Plug-in.
    * Speichern Sie die Datei DirectX9Display.rhp an einen geeigneten Ort auf Ihrem Computer (vielleicht im Rhino Plug-ins-Verzeichnis)
    * Starten Sie Rhino und führen Sie den Befehl PlugInManager aus. Klicken Sie auf die Schaltfläche Installieren und wählen Sie die Plug-in-Datei DirectX9Display.rhp aus.

=====Betriebssysteme=====

**Windows 2000/XP** - Das Plug-in sollte mit der SR0 von Rhino 4.0 funktionieren

**Windows Vista** - Das Plug-in sollte mit der SR0 von Rhino 4.0 funktionieren

=====Verwendung des Plug-ins=====

Wenn das Plug-in installiert ist, werden Sie einige Anzeigemodi benötigen, die DirectX verwenden.

  * Gehen Sie zu den Optionen unter Weitere Einstellungen > Erscheinung und erzeugen Sie eine Kopie einer Ihrer Anzeigemodi. Zum Beispiel;  Wählen Sie Schattiert aus, kopieren Sie diesen Modus und benennen Sie die Kopie auf etwas ähnliches wie Schattiert DX9 um. Zum Testen habe ich Kopien der Anzeigemodi Gitternetz, Schattiert und Gerendert verwendet
  * Gehen Sie zu den Einstellungen für den kopierten Anzeigemodus und wählen Sie Weitere Einstellungen aus. Wechseln Sie die Pipeline auf DirectX 9.
Wenn Sie eines Ihrer Ansichtsfenster auf den neuen Anzeigemodus einstellen, sollten Sie Geometrie anhand von DirectX betrachten können (für die erste Release dieses Plug-ins wird DirectX 9 in der unteren linken Ecke des Ansichtsfensters angezeigt.)

===== Befehle=====

**DirectX9Options** - Optionen zum Anzeigen von DirectX Ansichtsfenstern. Die Optionen sind

  * TransformLighting - definiert, an welcher Stelle bestimmte Berechnungen vorgenommen werden (in der Software oder auf der Grafikkarte). Änderung dieser Einstellung sollte Einfluss auf die Leistung haben.
  * AntiAliasCurves - Linien und Kurven können mit Anti-Alias Effekt gezeichnet werden. Der Nachteil beim Antialiasing ist die Anzeigegeschwindigkeit. Diese Option erlaubt Ihnen, nie Anti-Aliasing zu verwenden, immer Anti-Aliasing zu verwenden oder nur dann zu verwenden, wenn das Ansichtsfenster unbewegt ist.
