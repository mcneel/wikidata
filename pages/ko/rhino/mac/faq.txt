====== Mac용 Rhino 질문과 대답(FAQ) ======

다음은 Mac용 Rhino와 관련하여 가장 많이 물어보시는 질문과 그에 대한 대답니다. 여러분의 질문이 여기에 없다면 [[http://discourse.mcneel.com/c/rhino-for-mac|Mac용 Rhino 포럼]]을 방문하셔서 이미 답변이 올라온 질문인지 확인해 보세요.

==== 일반 ====

-->Mac용 Rhino 5는 언제 출시됩니까?#

곧 출시될 예정입니다. [[http://discourse.mcneel.com/t/rhino-for-mac-release-candidate-1/20047|이 글]]을 참조하세요.

<--

-->언제 릴리스되나요? 가격은 얼마인가요?#

아직 최종 가격을 결정하지 않았으나, Windows용 Rhino 5 보다는 저렴할 것입니다. Windows용 Rhino 5 소유자를 대상으로 사전 릴리스 특별 행사 가격이 제공되고, 한정된 기간 동안 모든 분들을 대상으로 릴리스 직후 특별 가격으로 판매될 예정입니다.

<--

-->Windows용 Rhino를 구입하면 Mac용 Rhino 5를 무료로 사용할 수 있나요?#

아닙니다. 하지만, Windows용 Rhino 5 소유자들은 사전 릴리스 프로모션 할인을 적용받으실 수 있습니다. Windows용 Rhino 5 라이선스 1개에 Mac용 Rhino 5 라이선스 1개를 특별한 가격으로 구입하실 수 있습니다.

<--

-->Mac용 Rhino 5 가 릴리스된 후, 서비스 릴리스나 업데이트가 있을 예정입니까?#

예. Mac용 Rhino 5의 업데이트에는 오류 수정, 새 기능 등이 포함됩니다. 처음 릴리스에는 레이아웃, 작업세션과 같은 기능이 포함되어 있지 않지만, 차후 업데이트를 통해 제공될 예정입니다. Mac용 Rhino 5를 구입하신 분은 추가 비용 없이 업데이트를 이용하실 수 있습니다.

<--

-->Mac용 Rhino와 Windows용 Rhino는 어떻게 다릅니까?#

  * Mac용 Rhino의 인터페이스는 Windows용 Rhino와 확연히 다릅니다.
  * Mac용 Rhino 5에서는 아직 플러그인이 지원되지 않습니다(곧 지원 예정에 있스니다).  [[http://discourse.mcneel.com/t/when-will-plugins-be-available-in-rhino-for-mac/20369|자세히...]]
  * Mac용 Rhino 5에는 아직 Windows용 Rhino의 모든 기능이 포함되어 있지 않습니다.  [[http://www.rhino3d.com/mac-feature-compare|자세히...]]
  * Mac용 Rhino 5에는 Windows용 Rhino의 모든 명령이 있는 것은 아닙니다.  [[http://wiki.mcneel.com/ko/rhino/mac/rhino5wincommandsnotinrhino5mac|자세히...]]

<--

-->Mac용 Rhino는 영어 이외의 언어로 사용할 수 있습니까?#

예. 현재 한국어, 스페인어, 프랑스어, 독일어, 이탈리아어, 중국어(간체와 번체), 일본어가 지원됩니다.  OS X의 기본 언어가 Mac용 Rhino에서 사용됩니다. 

OS X의 기본 언어와 다른 언어를 사용하려면 응용 프로그램 메뉴에서 다음과 같이 설정하세요:
  - **Rhinoceros** > **환경설정...** > **언어**로 갑니다.
  - **언어** 패널의 **Rhino 언어**에서 원하는 언어를 선택합니다.

<-- 

-->WIP는 무엇입니까?#

WIP는 Work In Progress(개발 중)을 뜻합니다. 일반적인 "beta"의 의미입니다. 안정적인 상업용 릴리스와 평행하게 릴리스되지만, WIP에는 실험적인 또는 아직 미완성인 기능이 포함되어 있습니다. WIP는 소프트웨어의 상업용 (또는 교육용) 버전을 소유하고 계신 분들께서 사용하실 수 있습니다. WIP는 불안정하거나, 오류가 있을 수 있으며, 미완성인 기능이 탑재되어 있습니다.
<--

--> WIP에는 시간 제한이 있습니다. 만료된 후에는 어떻게 되나요?#

만료일을 설정함으로써, 모든 사용자가 최신 버전을 테스트하고 이미 수정된 문제를 다시 보고하지 않게 됩니다. Rhino WIP 버전이 만료하기 전에 새로운 Rhino WIP가 공개되므로 사용자는 이를 [[http://www.rhino3d.com/download/rhino-for-mac/5B/wip|다운로드]]하여 사용하실 수 있습니다. Rhino WIP는 시작할 때마다 업데이트를 확인하고, 새 버전이 있으면 이를 사용자에게 알려 드립니다. 새로운 Rhino WIP 버전이 공개될 때마다 [[http://discourse.mcneel.com/|Discourse 포럼]]의 [[http://discourse.mcneel.com/category/rhino-for-mac|Mac용 Rhino 카테고리]]에도 안내됩니다.

==== 호환성 ====

-->Mac용 Rhino의 3DM 파일이 Windows용 Rhino에서 호환이 됩니까?#

예. Rhino 5 파일은 동일합니다. Rhino 파일을 Windows용 Rhino 4용으로 내보낼 수 (export) 있습니다.

<-- 

-->Mac용 Rhino에서 무슨 파일 형식을 지원합니까?#

꽤 됩니다. [[http://www.rhino3d.com/formats|지원되는 파일 형식 목록]]을 참조하세요.

<--

-->단추가 2개 또는 3개인 마우스를 사용할 수 있습니까?#

예. 모든 Mac 컴퓨터는 여러 개의 단추와 스크롤 휠이 있는 마우스를 지원합니다. 단추가 1개인 Mac 마우스도 사용하실 수 있습니다. Control 키를 누른 상태에서 단추를 클릭하면, 마우스 오른쪽 단추 클릭처럼 실행됩니다.

<-- 

-->Spaceball/SpaceNavigator/SpaceExplorer를 사용할 수 있나요?#

현재 SpaceNavigator만 지원됩니다.

<--

--> Mac용 Rhino에서 디지타이저가 지원됩니까?#

아니요. 현재 지원되지 않습니다.

==== 스크립트와 플러그인 ====
-->Mac용 Rhino에서 명령 스크립트를 실행할 수 있나요?#

예. 명령 스크립트를 로드하고 실행하려면 Rhino 메뉴에서 **도구 > 명령 > 파일에서 읽기**를 선택합니다.
 
<--

-->Mac용 Rhino에서 RhinoScript 프로그램을 실행할 수 있나요?#

아니요. RhinoScript는 VisualBasic을 기반으로 하며, VisualBasic은 OS X에서 실행되지 않습니다. Mac과 Windows 버전 모두에서 사용하는 스크립팅에 Python을 사용하고 있으며, 두 버전의 Rhino에서 스크립트를 작성할 수 있는 방법을 제공하고 있습니다. [[http://discourse.mcneel.com/category/scripting | Rhino 스크립팅 지원 포럼]]에서 Windows와 Mac 모두를 위한 Python 스크립팅 지원을 참조하세요.

<--


-->Mac용 Rhino에서 서드 파티 (third-party) 플러그인이 실행됩니까?#

아마도 그럴 겁니다. 하지만, Windows용 플러그인은 Mac에서 실행되지 않습니다. Mac용 Rhino로 플러그인을 포트하는지의 여부는 해당 플러그인 개발자가 결정할 사안입니다.
<--

-->제가 작성한 .NET 플러그인이 Mac용 Rhino에서 지원됩니까?#

지원 가능하도록 현재 작업 중입니다. Rhino 플러그인 개발자들이 OS X와 Windows에서 모두 실행되는 플러그인을 작성할 수 있게 만드는 툴을 당사에서 개발 중입니다.

<--

-->Mac용 Rhino가 제가 작성한 C/C++ 플러그인을 지원합니까?#

앞으로 지원될 예정입니다. 아직 C 플러그인에 대한 사용자 지원을 할 준비가 되지 않았으나, 앞으로 지원되도록 계획 중입니다. 사용자가 Xcode 개발 시스템을 사용하여 해당 플러그인을 다시 컴파일해야 합니다. 해당 플러그인의 복잡한 정도에 따라, Windows와 Mac 모두에서 컴파일되고 실행되는 플러그인을 만드는 데 시간이 다소 소요될 수 있습니다.

-->OS X 에서 ""다음 디스크 이미지를 열 수 없습니다." 어떻게 해야 합니까?#

아마도 다운로드한 .dmg 파일이 손상된 것으로 보입니다. 다운로드하신 .dmg 파일이 유효한지 확인하세요. 간혹, 다운로드하신 파일이 손상되어 있는 경우가 있습니다.

  - 디스크 유틸리티 프로그램 **/응용 프로그램/유틸리티/디스크 유틸리티** 을 시작합니다.
  - 메뉴에서 **이미지 > 검사...**를 선택하면 검사하려는 이미지 선택 대화상자가 표시됩니다.
  - 다운로드한 .dmg 파일을 선택합니다. 일반적으로 다운로드한 파일은 Rhinoceros_5A123.dmg 와 같은 형식의 이름을 가졌으며, 다운로드 폴더에 있습니다
  - 다운로드한 파일이 유효하지 않은 경우, [[http://www.rhino3d.com/download/rhino-for-mac/5/evaluation|다운로드]]를 클릭하고, 새로운 다운로드 링크를 요청하세요.
  - 이러한 현상이 다시 발생한다면, 귀하께서 사용하시는 인터넷 연결이 불안정하기 때문이므로, 다른 인터넷 연결에서 다시 파일을 다운로드하셔야 합니다. 다운로드 파일은 100MB 이상입니다.

<-- 

-->Rhino 최신 버전을 설치하였으나 만료됐다는 메시지가 표시됩니다. 어떻게 해야 합니까?#

귀하의 컴퓨터에는 2개 이상의 Rhino가 설치되어 있으며, 만료된 버전부터 시작하므로 해당 메시지가 표시됩니다. **모든** Rhino를 제거하셔야 합니다. 

아무 3DM 파일을 두 번 클릭합니다. Rhino에서 파일이 열리면 어딘가에 Rhino가 설치되어 있음을 뜻합니다. 
  - Dock에서 Rhino 아이콘을 두 번 클릭하고 메뉴에서 **옵션 > 파인더에서 표시(Show in Finder)**를 선택하여 위치를 찾습니다.
  - 찾을 Rhino를 Trash로 드래그합니다. 
  - 더 이상 Rhino가 없을 때, 3DM 파일을 두 번 클릭하면 파일이 TextEdit 에서 열립니다. 이렇게 되면 이제 모든 Rhino가 제거되었습니다.
  - 방금 다운로드하신 Rhino 버전을 다시 설치하세요.

<--


-->Mac용 Rhino는 어떻게 설치 제거합니까?#

다음은 사용자의 컴퓨터에서 Mac용 Rhino를 완전히 제거하는 방법입니다:

  - Application 폴더에서 Rhino를 제거합니다.
  - **/Library/Application Support/McNeel** 디렉터리를 제거합니다. 이 디렉터리에는 튜토리얼 모델과 메뉴얼(사용자가 다운로드했었다면)이 있을 수 있습니다. 이 폴더에는 사용자의 라이선스 파일이 있습니다.
  - 홈 디렉터리에서 **~/Library/Application Support/McNeel** 디렉터리를 제거합니다. 이 디렉터리는 Rhino에서 선택적으로 사용되며, 사용자의 컴퓨터에 없을 수도 있습니다. 사용자가 만든 재질, Python 스크립트, 기타 리소스가 있을 수 있습니다.
  - 홈 디렉터리에서 **~/Library/Preferences/com.mcneel.rhinoceros.plist** 파일을 제거합니다. 이 파일에는 사용자의 모든 Rhino 환경설정이 포함되어 있습니다.

--> 오른쪽 클릭으로 마지막 명령이 반복되었으면 좋겠습니다. 왜 안 되나요? #

상황에 맞는 메뉴를 해제하면 마우스 오른쪽 클릭으로 반복 작업을 실행할 수 있습니다.

  - 메뉴에서 **Rhinoceros**를 클릭하고 **환경설정**을 클릭합니다.
  - **마우스** 패널을 선택합니다.
  - **마우스 오른쪽 단추** 항목에서 **상황에 맞는 메뉴 사용** 확인란을 선택 해제합니다.

--> Paneling Tools를 사용할 수 있나요? #

예. PanelingTools 플러그인은 릴리스된 Mac용 Rhino 5에 포함되지 않습니다. 하지만 **[[http://www.rhino3d.com/go/download/rhino-for-mac/wip/latest|Mac용 Rhino WIP]]**에서 사용하실 수 있습니다. Mac용 Rhino 라이선스를 갖고 계시다면 Mac용 Rhino WIP을 다운로드하셔서 Paneling Tools를 사용하실 수 있습니다. Mac용 Rhino WIP 는 개발 중(Work-In-Progress)이므로, 최신 기능이 있으나 아직 안정성이 떨어지고 오류가 있을 수 있습니다. 기능이 추가되고 오류가 수정됨에 따라 새로운 WIP가 자주 릴리스됩니다. WIP에는 Paneling Tools 메뉴가 없으며, 모든 명령은 명령행을 통해서만 사용하실 수 있습니다. 모든 Paneling Tools 명령을 보시려면 명령행에서 “pt”를 입력하여 자동 완성된 명령 목록을 참조하세요.
