======Comment déterminer le nom de l'hôte du Zoo======
> **Produit:** //[[zoo:zoo5|Zoo 5]]//
> **Résumé:** //Indique comment déterminer le nom d'hôte du système où est installé le Zoo.//

=====Question=====
J'ai acheté deux copies de Rhino 5pour les partager sur 5 systèmes. Nous avons donc installé le Zoo et ajouté les deux licences de Rhino. 

Après avoir installé Rhino, j'ai essayé de le démarrer. Dans la boîte de dialogue qui apparaît, je clique sur l'option **Détecter le serveur Zoo automatiquement** puis je clique sur le bouton **Tester la connectivité**. 

{{:zoo:zoo_host0.png}}

Mais, Rhino me dit juste qu'il ne peut pas détecter mon serveur Zoo. Que dois-je faire ?
=====Réponse=====
Si Rhino ne peut pas détecter automatiquement votre serveur Zoo, car vous n'avez pas ajouté d'entrée pour le [[http://wiki.mcneel.com/zoo/findzooserver|serveur Zoo par "défaut" ]] sur le serveur DNS de votre entreprise; indiquez manuellement le nom d'hôte ou l'adresse IP du système où tourne le Zoo.

=====Plus d'informations=====
Si vous n'êtes pas sûr du nom d'hòte du système où se trouve le serveur Zoo, voici quelques astuces pour le trouver. 

====1. Lancer la commande Hostname====
Sur le système où se trouve le serveur Zoo, ouvrez une fenêtre d'invite de commandes. Lancez la commande **Hostname**. 

Par exemple :

{{:zoo:zoo_host1.png}}

Dans cet exemple, le nom d'hôte est "zoo". 

====2. Lancer ZooAdmin=====
Sur le serveur Zoo, lancer l'utilitaire d'administration du Zoo, ou **ZooAdmin.exe**. La barre de titre de la fenêtre de l'administrateur du Zoo affichera le nom d'hôte, avec le nom de domaine, du système sur lequel le Zoo est installé. 

Par exemple :

{{:zoo:zoo_host2.png}}

Dans cet exemple, le nom d'hôte est "zoo.mcneel.com".
