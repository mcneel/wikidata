====== 도구모음이 사라지는 문제 ======

> **Summary:** //도구모음이 사라지는 문제의 원인과 해결 방법을 소개합니다.//

======도구모음이 사라지는 문제======

**Rhino의 도구모음**은 .tb 확장자를 가진 외부 파일에 저장되어 있습니다 (이 파일은 때때로 작업공간 파일이라고도 불립니다). Rhino 를 켰을 때 도구모음이 사라졌다면 대부분의 경우 Rhino 에서 도구모음 파일로의 경로를 "잃어버렸기" 때문입니다. 원인은 여러 가지일 수 있습니다. 다음 방법으로 도구모음을 복원하십시오:

**우선**, Rhino 4 와 기본 작업공간 (default.tb) 을 사용한다면, **ToolbarReset** 명령을 실행하시면 됩니다. 이 방법으로 문제가 바로 해결되시기를 바랍니다.

> **Note:** //default.tb 파일을 수정했다면, 원래 파일이 아니라 수정된 도구모음이 로드됩니다.//

**Rhino 4.0 에서 사용자가 지정한 작업공간을 사용하는 경우**(예전에 다른 이름으로 저장한 파일), ToolbarReset 명령을 실행하면 사용자가 원하는 파일이 아니라 default.tb 가 다시 로드됩니다. Rhino 3.0 에서는 ToolbarReset 명령이 실행되지 않습니다.

어떤 이유에서건 ToolbarReset 명령으로 원하는 결과를 얻지 못 할 때, 다음 과정을 실행해 보세요:

**도구 > 도구모음 레이아웃 (Tools > Toolbar Layout)** 에서 **파일 > 열기 (File > Open)** 를 선택합니다.

열린 대화 상자에서 사용자가 원하는 도구모음 파일을 찾을 수 없다면 (default 도구모음 레이아웃을 사용하고 있다면 default.tb 파일이 됩니다), C:\Documents and Settings\All Users\Application Data\McNeel\Rhinoceros\4.0  (또는 3.0) 경로를 찾습니다.

그 위치에서 //반드시// default.tb 파일을 찾을 수 있습니다. 같은 위치에 사용자가 지정한 작업공간 파일을 저장했다면 같은 위치에서 찾으실 수 있을 것입니다.

이후에도 계속해서 작업공간이 열리지 않으면 해당 파일이 손상되었을 수도 있습니다. 사용하고 있는 도구모음이, 수정하지 않은 기본 작업공간 (default.tb) 이라면, 같은 디렉터리에 default.tb.bak 파일을 찾을 수 있을 것입니다. Rhino 의 모든 창을 닫고, 손상된 파일과 해당 파일의 .bak 파일을 모두 제거한 후 다시 시도해 보세요.

여전히 문제가 계속되면, default_backup.tb 파일을 사용하는 응급 처치 방법이 있습니다. 이 파일의 복사본을 만들고 이름을 default.tb 로 지정합니다.

사용자 지정 작업공간을 사용하면서 적어도 한 번 이상 Rhino 를 열고 닫았다면 해당 도구모음 파일의 .bak 파일이 있을 것입니다. 만약 그렇지 않다면, 다른 곳에 저장한 사용자 지정 작업공간의 백업 복사본을 찾아야 합니다. 백업 복사본을 만들어 두셨겠지요?

> **ADVICE:** //기본 작업공간 (**default.tb**) 을 어떻게든 수정했다면 새 이름으로 저장하시길 강력하게 권합니다. 새로 설치하거나 Rhino 를 업데이트하게 되면 **default.tb** 이 새 파일로 덮어쓰여지게 되어 사용자가 지정해 둔 **모든** 설정이 사라지게 됩니다. **도구 > 도구모음 레이아웃** 에서 **파일 > 다른 이름으로 저장** 을 선택하고 수정된 작업공간의 이름을 지정합니다. 항상 사용자가 지정한 작업공간의 변경 내역을 자신만의 작업공간 파일에 저장하시기 바랍니다. 이 파일을 다른 곳에도 저장하여 백업 파일로 보관하십시오. (다른 중요한 문서 파일과 마찬가지로 다른 컴퓨터에도 저장하시기 바랍니다.) **만약 이 파일을 잃어버리면 다시 만드는 데 얼마나 많은 시간이 들어갈 지 생각해 보세요.**//

> **IMPORTANT:** //**도구모음/작업공간을 수정할 때는 반드시 Rhino 창을 하나만 켜두셔야 합니다!**  도구모음 설치 후, 도구모음 레이아웃 대화상자에서 파일>저장 메뉴를 사용하여 변경된 내역을 저장합니다. 또는 Rhino 를 종료하고 다시 시작하십시오. Rhino 는 종료시 도구모음 변경 내역을 자동으로 저장합니다. Rhino 를 여러 창으로 열어 두면 마지막으로 종료된 Rhino 의 도구모음 설정이 앞서 저장된 도구모음을 덮어쓰게 되므로 사용자가 지정한 변경 사항이 없어질 수 있습니다.//

======도구모음 고정 문제======
====Rhino 3.0 에서 도구모음 레이아웃이 고정되지 않거나 올바르게 고정되지 않는 문제====

Rhino 3.0 을 닫습니다.

Rhino를 종료한 후, 이 유틸리티를 실행하세요: http://en.wiki.mcneel.com/content/upload/files/Rhino3_Registry_Repair.zip

Rhino 를 다시 시작합니다.
