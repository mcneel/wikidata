====== Mac版Rhinoに関してよく寄せられる質問（FAQ） ======

--> Mac版Rhinoは購入できますか？#

いいえ。Mac版Rhinoは現在開発期間中で、販売はしていません。WIP（ワーク・イン・プログレス）版を無償でお試しいただけます。ダウンロードは[[http://www.rhino3d.com/download/wenatchee/5.0/wip|こちらから]]行っていただけます。フィードバックをお待ちしています。

<--

--> Mac版Rhinoには使用期限が設定されています。期限が切れるとどうすればよいのですか？#

使用期限が設定されているのは、テストしていただいているバージョンが最新で、既に修正された問題（バグ）を報告していただく必要がないことを確実にするためです。

それぞれのMac版RhinoのWIPバージョンの期限が切れる前には、 新しいWIPが[[http://www.rhino3d.com/download/wenatchee/5.0/wip|ダウンロード]]できるようになります。Mac版Rhinoは起動時にいつも新しいWIPがあるかどうかを確認します。そしてもしある場合は、それを知らせます。お知らせは[[http://discourse.mcneel.com/ |Discourseフォーラム（英語）]]の[[http://discourse.mcneel.com/category/rhino-for-mac|Mac版Rhinoのセクション]]でも行われます。

<-- 

-->データファイルはWindows版と互換性はありますか?#

はい、Rhino 5のファイルはまったく同じです。Windows版のRhino 4で使用できるようにRhino 4のファイルもエクスポート可能です。

<-- 

-->Macintosh版で2つのボタン/3つのボタンのあるマウスを使用できますか?#

はい、すべてのMacintoshのコンピュータは複数のボタンそしてスクロールホイールのあるマウスをサポートしています。もちろんMacintoshの1ボタンのマウスも使用できます。右マウスボタンのクリック操作を行うには、Controlキーを押しながら（1つの）ボタンをクリックします。
<-- 

-->Macintosh版でSpaceball/SpaceNavigator/SpaceExplorerを使うことはできますか?#

現在はSpaceNavigatorのみがサポートされています。

<--




-->動作しない機能があるのはなぜですか? 「不明または実装されていないコマンド（Unknown or unimplemented command）」と表示されるのはどうしてですか?#

私共は現実に可能な限りMacintoshのプラットフォームにWindows版Rhinoと同じ機能を搭載する計画をしています。しかし、このMacintoshのWIP版には搭載予定のすべての機能はまだ実装されていません。Windows版Rhinoの[[http://www.rhino3d.com/nr.htm|ソフトウェア開発のプロセス]]をご覧ください。Macintosh版の開発も同じ要領で行っていきます。Macintosh版の現在の段階は、[[http://www.rhino3d.com/nr.htm#phase1|フェーズ1: ワーク・イン・プログレス]]です。 

現在動作している機能、そしてまだ実装が必要な機能については、[[rhino:mac:whatworks|こちらのウェブページ]]をご覧ください。

<-- 

-->Macintosh版では英語以外の言語はサポートされていますか?#

はい、現在の段階では、スペイン語、フランス語、ドイツ語、イタリア語、中国語、日本語、そして韓国語がサポートされています。（現在はまだWIPバージョンですので、これらの言語版でも英語で表示される部分があります。） 

Macintosh版RhinoはMacintoshにビルトインされたデフォルトの言語を使用します。OS Xのデフォルトとは異なる言語を使用されたい場合は、アプリケーションメニューで下のことを行ってください。
  - メニューで**Rhinoceros**をクリックし、**Preferences**をクリックします。
  - **Language Preference**パネルで希望される言語を選択します。

<-- 
-->Rhinoをアンインストールするにはどうしたらよいですか?#

下の手順でお使いのコンピュータからMac版Rhinoを完全にアンインストールすることができます。

  - Applicationsフォルダでアプリケーションを削除します。
  - ディレクトリ「**/Library/Application Support/McNeel**」を削除します。このディレクトリには、（ダウンロードしていた場合）チュートリアルモデルとマニュアルが保存されています。（ダウンロードしていない場合はこのディレクトリは存在しません。）
  - ホームディレクトリの「**~/Library/Application Support/McNeel**」を削除します。このディレクトリはRhinoによって随意に使用されるためお使いのコンピュータに存在しない可能性があります。このディレクトリには、（作成されていた場合）Toucanのマテリアル、Pythonスクリプト、そして他のMac版Rhinoのプラグインがある場合があります。
  - ホームディレクトリのファイル「**~/Library/Preferences/com.mcneel.rhinoceros.plist**」を削除します。このファイルにはすべてのRhinoの設定（Preferences）が含まれています。

アップグレードのポリシーについてはまだ何も決まっていません。

-->Mac版RhinoはWindows版Rhinoと何が違うのですか?#
  * Mac版RhinoにはWindows版Rhinoのほぼすべてのモデリング機能が搭載されます。
  * ビルトインレンダラ、寸法、テキストなどの注釈オブジェクト、シンプルな製図機能が搭載されます。

__Mac版Rhinoは下の点でWindows版Rhinoと異なります。__
  * プラグインやアドオン機能で拡張できません。
  * したがって、RhinoCAM、RhinoGold、Matrix、Grasshopper、もしくはどのようなアドオンも追加できません。
  * レイアウトは搭載されません。

Mac版Rhinoの最初のリリースは、すべてのユーザーの方向けという訳ではありません。特に作業にプラグインが必要なユーザーの方には、最初のリリースは向いていません。  
しかし、多くのユーザーの方には必要な機能が揃っていると思います。
-->

==== スクリプトとプラグイン ====

-->Macintosh版でコマンドスクリプトは実行できますか?#

はい。コマンドスクリプトをロード、実行するには、Rhinoメニューの**ツール > コマンド > ファイルから読み込み**を選択します。
 
<--

-->Macintosh版でRhinoScriptプログラムは実行できますか?#

いいえ。RhinoScriptはVisualBasicに基づいており、VisualBasicはOS Xで動きません。MacとWindows版両方でのスクリプティングはPythonを使っています。この方法で両方のバージョンのRhinoのスクリプト作成が可能になります。WindowsおよびOS X両方のPythonスクリプトのサポートに関しては、[[http://discourse.mcneel.com/category/scripting | Rhinoスクリプティングサポートフォーラム]]をご覧ください。

<--


-->Mac版Rhinoではサードパーティのプラグインは使える予定ですか?#

おそらくそうだと思います、しかし、WindowsのプラグインはMacでは動作しません。プラグインをMac版Rhinoに移植するかは、それぞれのプラグイン開発会社が決定することです。
<--
 
-->Macintosh版で私が作成したCプラグインは動作する予定ですか?#

そのうち予定しています。現在はCプラグインのユーザーサポートを行う準備ができていませんが、将来は予定しています。その際はMacintoshの開発システムを使用してプラグインを再コンパイルする必要があります。プラグインがどのくらい複雑かによりますが、WindowとMacintoshitの両方でコンパイル、動作するプラグインを作成するのはたいした作業量でない可能性があるでしょう。

<--

-->Macintosh版で私が作成した.NETプラグインは動作する予定ですか?#

そのうち予定しています。現在Rhinoのプラグイン開発者がOS XとWindowsの両方で実行できるプラグインを書けるツールを準備中です。

==== インストールに関する問題 ====

-->新しいバージョンをインストールする前に、Rhinoをアンインストールする必要はありますか?#

ありません。 

<-- 

-->OS Xにより「The following disk images couldn't be opened.」と表示されます。どうしたらよいですか?#

ダウンロードされた.dmgファイルが破損している可能性があります。ダウンロードされた.dmgファイルが有効か確認してください。ファイルはダウンロード時に時として破損することがあります。

  - ディスクユーティリティプログラムを開始します。（**/Applications/Utilities/Disk Utility**）
  - メニューから**Images > Verify...**を選択します。Open Fileダイアログボックスが表示されます。
  - ダウンロードした.dmgファイルを選択します。ファイルは通常Downloadsフォルダにあり、rhinoceros_wip_YYYYMMDD.dmg（YYYYは年、MMは月、DDは日）という名前が付いています。（例: rhinoceros_wip_20141023.dmg）
  - ダウンロードしたファイルが有効でない場合、[[http://www.rhino3d.com/download/wenatchee/5.0/wip|ダウンロードサイト]]から再度ダウンロードしてください。
  - 同じ問題が起こる場合、お使いのインターネット接続が不安定である可能性があります。別のインターネット接続でダウンロードをお試しください。ダウンロードされるファイルは、100 Mb以上のサイズです。

<-- 

-->最新バージョンのRhinoをインストールしたのに、期限が切れていると表示されます。どうすればよいのでしょうか?#

複数のRhinoのコピーがインストールされているため、期限が切れているコピーがまだ起動される状態になっています。Rhinoの**すべての**コピーをアンインストールしてください。 

3DMファイルどれか一つをダブルクリックしてみてください。Rhinoでファイルが開かれた場合、Rhinoのコピーが存在しています。 
  - DockのRhinoのアイコンを右クリックし、メニューから**Options > Show in Finder**を選択して場所を調べます。
  - 見つかったコピーをゴミ箱にドラッグします。 
  - Rhinoのコピーがまったくない状態になると、3DMファイルはダブルクリックするとテキストエディタで開かれます。このような状態になれば、Rhinocerosのコピーはすべて取り除かれています。
  - ダウンロードしたばかりの最新のバージョンを再度インストールしてください。

<--
 
-->RhinoをApplicationsフォルダにドラッグすると、BGHUDAppKitが実行されているのでインストールできませんと表示されます。どうすればよいですか?#

これは10.7 Finderのバグです。このバージョンのFinderは、コピーを開始する前にアプリケーションを上書きできるかどうかを正しくチェックしません。このエラーが表示される可能性、またはInfo.plistについてのメッセージが表示される可能性があります。

あいにくこのメッセージが表示される場合、Finderは既に既存のRhinocerosのアプリケーションを部分的に取り除いており、また新しいバージョンをまだインストールしていません。This means your current copy of Rhinoceros is damaged and will crash if you try to run it.

If you try the drag and drop a second or third time, it eventually works. You can also drag your existing copy of Rhinoceros from the Applications folder to the Trash before installing a newer version. Then the Finder won't have a problem trying to overwrite the existing version.
 
The 10.8 Finder does this correctly. Upgrading to 10.8 is another "workaround".

-->I got a mysterious dialog that said "Program Failure" and now Mac Rhino does not work any more.#

When Mac Rhino encounters an unexpected situation, it displays a dialog with a generic "Program Failure" message. After this error, Mac Rhino shuts down internally, and you cannot run commands or menu choices after this point. You can save your file, and you can quit Mac Rhino, but you won't be able to do more. We need to know when this happens. 
When working with Rhino, you may see the following dialog:

Rhino displays this generic dialog for several situations. Not every feature is implemented yet in Rhino, and you will get this dialog when you attempt to use an unimplemented feature. There is also program code in Rhino that we expect should only be used in the Windows version of Rhino. If something you do tries to execute this code, you will also get a "Rhino Error" dialog. Lastly, there are lots of debugging tests in this early code, and you will see this dialog when an internal test discovers a situation that should not happen.

{{:rhino:mac:notimpl.png?nolink|}}

When you encounter one of these dialogs, write down the file name, line number, and version information from the dialog. Email this information, along with a description of what you were doing when you received the dialog, to us at <mactech@mcneel.com>. We will try to duplicate your problem and find a solution so that this error will not happen in the future.

After receiving this dialog, Rhino restricts you to just a few commands. You'll be able to save your file and you can quit Rhino, but not much else will work.

--> I want right click to repeat the last command #

Disabling the Context menu causes allows the repeat behavior.

  - On the menu, click **Rhinoceros** and then click **Preferences**.
  - Select the **Mouse** panel.
  - In the **Right Mouse Button** section, clear the **Enable Context Menus** check box.

<--

--> I want right click to execute the secondary command in a tool button#

Some tool palette buttons have two commands attached to the button. The commands are listed in the tooltip.
The default action to execute the alternate command is to Option+click the tool button.

  - On the menu, click **Rhinoceros** and then click **Preferences**.
  - Select the **ToolPalette** panel.
  - You want to change the **Execute alternate command when** setting. To change it to a right click, put the cursor over the box icon to the right of the **Execute alternate command when** setting.  Click the right mouse button.  The prompt changes to **Right click**.
