====== nXt Bildeditor ======

Der nXt-Bildeditor ist ein leistungsstarkes, eigenständiges Hilfsprogramm, das native Bilddateien (.nXtImage), die von einer beliebigen nXt-Plattform erzeugt werden, bearbeiten kann.  Diese nativen Dateien enthalten die gesamte Information, die während eines Renderings gesammelt wird. Starten Sie den Editor im Windows Startmenü > nXt-Ordner.

Mit dem Materialeditor können Sie:

  * die Einstellungen des Tonreglers anpassen
  * die Intensität eines beliebigen Beleuchtungskanals ändern
  * besondere, bildgestützte Effekte hinzufügen, wie Dunst oder Tiefenschärfe
  * ein Tone-Mapping-Bild in ein bekanntes Format wie .jpg oder .png speichern
  * die Luminanzinformationen in ein HDR-Format speichern
  * zusätzliche maskierte Kanäle (Alpha, Abstand, Material) zur Verwendung in fortgeschrittenen Kompositionen visualisieren und speichern
  * ein Piranesi© Dateiformat (*.epx) speichern, das zur Erzeugung von nicht-fotorealistischen Renderings verwendet werden kann
  * Bildarithmetik für Aufgaben wie Zusammenfügen eines Bilds, das aus verschiedenen Knoten auf der Renderfarm erzeugt wurde, verwenden.
  * die Beleuchtungseinstellungen speichern, die für ein Rendering verwendet wurden.  Diese Beleuchtungseinstellungen können verwendet werden, um weitere Renderings zu erzeugen.


===== Bild am Laden =====

Nachdem Sie Ihr Rendering als .nXtImage gespeichert haben, gehen Sie im Menü Datei auf Öffnen, um das Bild in den Editor zu laden.  

Während Sie den Cursor über das Bild bewegen, zeigt die Statuszeile die folgende Information über jedes Pixel an: 

  * Die Pixelkoordinate, von der unteren linken Ecke gemessen
  * Erhaltene Farbe und Alpha im RGBA-Format des Pixels nach dem Tone-Mapping
  * Der tatsächliche Luminanzwert, in jedem Pixel als RGB und einzelner Y-Kanal gespeichert.  Dies ist das direkte Resultat der Beleuchtungsberechnung.  Wenn genaue Beleuchtungsinformationen verwendet wurden, um das Rendering zu erzeugen, werden diese Werte in Candella/m^2 ausgedrückt.
  * Abstand jedes Pixels zum Betrachter in Metern.  Negative Werte deuten auf einen Hintergrundpixel.
  * Name des Materials, das für das Rendering des Pixels verwendet wurde.


===== Arbeiten mit Maskenkanälen =====

nXtImage-Bilder enthalten drei zusätzliche Kanäle, die in den meisten Bitmap-Editoren als Masken für fortgeschrittene Komposition verwendet werden können.  Diese Kanäle enthalten Alpha- (Transparenz), Abstands- und Materialinformationen für jedes Pixel, in einem Grauskalabild codiert.  Jeder Kanal kann in eine .png Datei gespeichert und visualisiert werden.

Verwenden Sie das Pulldown-Menü Ansicht, um jeden Maskenkanal zu visualisieren.
Verwenden Sie die Option unter Datei->Maske speichern, um einen Maskenkanal als .png Bild zu speichern.

**Zusätzliche Bemerkungen:**

  * Der Alpha-Kanal kann mit einem Tonemapping-Bild ausgestattet werden, indem ein Dateiformat mit Alpha ausgewählt wird, wenn ein Tonemapping-Bild gespeichert wird.
  * Abstands- und Materialkanäle verfügen über kein Antialiasing und können hartkantige Artefakte aufweisen.  Wenn Sie vor Verwendung einer Maske etwas Gaußsche Unschärfe hinzufügen, könnte dies diese Kanten etwas weicher gestalten.
  * Der Materialkanal codiert nur 255 verschiedene Materialien.  Wenn Ihre Zeichnung über mehr Materialien verfügt, werden einige Maskenfarben wiederholt.
