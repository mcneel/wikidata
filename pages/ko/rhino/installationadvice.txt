**답변:** Rhino 5는 자동으로 32 비트 버전과 64 비트 버전을 설치하며, 각각의 바로 가기 아이콘이 생성됩니다. [[Why2Shortcuts|이 부분에 대한 자세한 설명은 여기를 참조하세요.]]

<color darkslategray>**//질문://** //왜 Rhino 5 설치 관리자에서 3 버전 2 버전 키 입력이 안 되나요?//</color>\\ 
**답변:**SR0 설치 관리자에 오류가 있었으나, 쉬운 해결 방법이 있습니다.\\
갖고 계신 2버전 또는 3버전 코드를 입력할 때 키보드의 CAPS Lock을 켜고 입력하시면 됩니다.\\

사용에 불편을 드려 죄송합니다.

=====새 컴퓨터에 Rhino 를 최신 서비스 릴리스로 설치하려면=====

**//질문://** //새 컴퓨터에 Rhino **X** 버전의 **Y** 서비스 릴리스로 설치하고 싶습니다. 저는 항상 Rhino 업그레이드 버전을 구매했습니다. 이제까지의 모든 이전 버전과 서비스 팩을 연달아 설치해야 합니까??//

**//답변://**  설치 CD 버전 중에 최신 업그레이드 버전만 설치하십시오. 설치 과정 중에 이전 버전의 CD 를 삽입하여 이전 버전의 유무를 확인하는 과정이 있지만, 이전 버전을 실제로 **설치**하실 필요는 없습니다. 일단 최신 설치 CD 버전을 설치하신 후, 최신 서비스 릴리스를 설치하십시오. 이전 버전의 서비스 릴리스를 설치하실 필요가 없습니다.

=====서비스 릴리스란?=====
이 설치관리자는 다운로드하실 수 있으며, 기존 제품이 설치되어 있어야만 설치가 진행됩니다. SR3 또는 SR4b와 같은 서비스 릴리스(SR) 번호와, 2005년 7월 29일자임을 뜻하는 20050729와 같은 다운로드 파일 이름으로 서비스 릴리스를 식별하실 수 있습니다.

=====설치된 버전이 어느 버전인지 어떻게 확인하나요?=====
소프트웨어를 설치하시면, 서비스 릴리스의 번호와 날짜를 볼 수 있습니다. 이 번호와 날짜는 Rhino 를 시작하면 "스플래시 화면"에 표시되며, Rhino에서 도움말 > [[rhino:rhinoceros|Rhinoceros]] 정보 메뉴에서도 확인하실 수 있습니다.

Flamingo에서는 FlamingoAbout 명령을 실행하세요. Bongo에서는 BongoAbout 명령을 실행하세요.

=====서비스 릴리스를 어디서 받을 수 있나요?=====
업데이트 확인 서비스를 사용하는 빠른 방법은 Rhino 의 도움말 메뉴에서 업데이트 확인 항목을 클릭하는 것입니다. 인터넷에 연결된 상태라면, 사용하고 계신 소프트웨어보다 최신 버전이 있는 경우 McNeel 웹 서버에서 올바른 서비스 릴리스를 다운로드하실 수 있도록 안내해 드립니다. Flamingo 에서는 Raytrace 또는 Photometric 메뉴에서 Check for Updates 를 선택하세요. Bongo 에서는 Bongo 메뉴에서 Check for Updates 를 선택하시면 됩니다.

====="프로그램 추가 또는 제거"로 이전 버전을 문제 없이 제거할 수 있습니까?=====
Windows의 제어판에서 "프로그램 추가 또는 제거" 항목은 모든 서비스 릴리스를 각각 250 MB 디스크 공간을 사용하는, 설치된 프로그램으로 보고합니다. 이것은 잘못 이해하기 쉽습니다. 서비스 릴리스를 설치하면 대부분 또는 모든 파일이 기존 설치 파일을 덮어쓰지만 Windows에서는 기존 설치된 파일에 새로운 파일이 추가된 것처럼 합계를 내어 보고하게 됩니다. Rhino 4.0의 이전 서비스 릴리스를 제거하기 위해 "프로그램 추가 또는 제거"를 **사용하지 마십시오**.

===== 오류 1327. 유효하지 않은 드라이브: =====
**//질문://** //설치할 때 "오류 1327. 유효하지 않은 드라이브 (E:, F:, 등)" 오류가 발생합니다.//

**//원인://** //사용자 계정에 하나 이상의 존재하지 않는 드라이브에서 "내 문서"와 같은 특별한 폴더를 사용해야 하는 설정이 있기 때문입니다.//

**//해결책://**
  - "시작" 메뉴에서 "실행"을 클릭합니다.
  - RegEdit 를 입력하고 "확인"을 클릭합니다.
  - HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Shell Folders 위치를 찾습니다.
  - 설치 관리자의 오류에 언급되어 있는 드라이버 글자가 포함되어 있는 디렉터리를 찾습니다. 이것을 올바른 드라이브와 폴더로 지정합니다.
  - HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders 위치를 찾습니다.
  - 설치 관리자의 오류에 언급되어 있는 드라이버 글자가 포함되어 있는 디렉터리를 찾습니다. 이것을 올바른 드라이브와 폴더로 지정합니다.

**안내:** 이 방법 또는 작업에는 레지스트리를 수정하는 방법이 포함되어 있습니다. 레지스트리를 잘못 수정하면 심각한 문제가 발생할 수 있습니다. 따라서 주의하여 이 과정을 실행하시기 바랍니다. 레지스트리를 수정하시기 전에 레지스트리를 백업하시면, 문제 발생 시 원래 레지스트리를 복구하실 수 있습니다. 레지스트리의 백업과 복구에 대한 자세한 정보는 Microsoft Knowledge Base의 다음 글을 클릭하셔서 참조하시기 바랍니다:
