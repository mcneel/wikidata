====== Mac上的Rhino for Windows ======

> **摘要：** 在 Intel 的 Mac 上執行各種 Rhino for Windows 版本的相關資訊。到 [[https://www.rhino3d.com/download/]]取得最新資訊並檢查作業系統需求。Rhino for Mac現在可以在Intel與 Apple Silicon Macs上執行。 

=====(舊) 資訊來自在Intel Macs上用Rhino for Windows的用戶=====

==新的 Parallels V5 運行得不太好==
事實證明，Parallels Desktop 5 中新的//加速圖形支援// 對於某些OpenGL 的應用程式（如 Rhino）來說成效相當差。
他們列出對 OpenGL 2.1 的支援，但許多用戶報告 Rhino 當掉。

==解決Parallels V5的問題==
我的 MacBook Pro（4GB RAM、雙核心、NVIDIA GeForce 8600M GT）上安裝了 Parallels Desktop 5 和 VMWare Fusion 3.1。
我還有一個 Bootcamp 分割區，上面有 Windows XP Pro。
為了讓 Parallels 與 Rhino 良好配合，請關閉 Rhino 中的硬體加速控制。
它在 //工具 - 選項 - 外觀 - OpenGL 下(Tools- Options - Appearance - OpenGL)//。這是最佳選擇。
至少 Rhino 可以使用且顯示驅動程式不再當掉。最終他們可能會解決這個問題，但我不會為他們通報自己應用程式的問題而付費。

==VMWare Fusion 3.1 運作得比較好==
VMWare Fusion (3.1) 的更新也增加了加速圖形支援。還包括 支援OpenGL 2.1 ，似乎可以與 Rhino 良好配合。
它對 Rhino 顯示性能進行了適度的改進。例如，如果您處於陰影顯示（或 EMap 等假彩色分析模式）中，並且您旋轉視圖，則從顯示中被遺漏的物件會比以前更少。這是一個雖不大但顯而易見的進步。

>**設定：** 如果你用的是滾輪滑滑鼠，請在VMWare Fusion 偏好設定中的鍵盤與滑鼠頁面，清除輔助按鈕(Secondary Button)以及按鈕捷徑(Button 3)的選項。 您才能夠像在Windows中一樣使用滑鼠右鍵。 另外在常規頁面(仍在Fusion偏好設定中)，將遊戲選項變更為 **永遠針對遊戲最佳化滑鼠**。這樣當滑鼠指標移動到您正在旋轉視圖視窗之外時，就不會發生瘋狂旋轉這種狀況。

==最佳解決方案==

儘管如此，它們的效能都無法與重啟到 Bootcamp 分割區並直接執行 Windows 的效能相比。這樣，您就可以使用所有記憶體、所有處理器核心，以及顯示卡提供的加速硬體。在我的 MacBook Pro 上，Rhino 的表現要好一個量級。 Bootcamp 是迄今為止最好的。VMWare Fusion 遠遠落後於 Fusion。 Parallels Desktop 則落後 Fusion 幾步，墊底。

//-John Brock 2010年6月30日30 June 2010//

**在 Bootcamp 下使用 ATI X1600 顯示卡的 MacBook Pro**

在這種情況下，前景恐怕不樂觀。
X1600 可能是 Windows 上 OpenGL 支援最糟糕的顯示卡之一，而且硬體配置也不算太好，也許這就是為什麼蘋果幾乎立刻就放棄了 X1600，轉而在其 MacBook 上使用 NVIDIA 顯示卡的原因。但問題不在這。請繼續閱讀。

對此我有親身經驗。問題不再於Rhino無法辨識ATI顯卡，而是Bootcamp驅動程式不支援也不提供ICD。
換句話說，Bootcamp 提供的驅動程式中沒有可用的硬體加速。
對於大多數在這些系統上執行Windows 的人來說，非常絕望，除非…

您可以取得名稱為  //MobilityDotNet//的公用程式來安裝經過修改的桌面驅動程式。 工作原理大致如下：

  - 將桌機版驅動程式下載到 MBP 上。 
  - 啟動驅動程式安裝過程。
  - 在驅動程式盪案解壓縮並複製到目標資料夾後，但在實際開始安裝**之前**(也就是安裝過程中暫停，ATI對話框詢問您要做什麼時)，執行MobilityDotNet公用程式。
  - 之後的步驟是修改相應的文件，以便驅動程式能夠識別該特定的 MBP 為 //有效// 的受支援硬體。現在您可以繼續安裝 ATI 驅動程式了。 
  - 需要重新啟動，之後 驅動程式就會隨著ICD安裝好了。

過去兩年多來，我一直在為我的 MBP 做這件事。但是，每次新發布的桌機驅動程式似乎都修復了一些問題，但又造成其他一些錯誤。我發現唯一能穩定運行的是 8.7版驅動程式，不過，那只適用XP 和 Vista 。我最近在我的 MBP 上安裝了 Win7，9.3 驅動程式似乎可以正常工作……但又得重複上述步驟。

因此，當您看到選取和取消選取 Rhino 的**使用硬體加速模式** 沒有任何作用的情況（即您總是看到 Microsoft Generic）時，通常意味著以下兩件事之一：

  - 通常是沒有安裝ICD。
  - 用戶在 Windows 顯示設定「故障排除」部分中調低了硬體加速設定。

抱歉，但針對這種情況，沒有其他答案或解決方案……我知道 iRhino 運行得挺不錯的，不過這顯然是因為操作系統原生支持硬體，而整個 OS X 系統都根據於 OpenGL，並依賴於加速……所以它最好能正常運作……而且確實也有。

唉，希望我有更好的方法。

//-Jeff 5 2010年 5月//

例如，您可以在一個可以最小化的視窗中執行 Internet Explorer 和 Microsoft Outlook 等 Windows 軟體，就像其他 Mac 程式一樣。
您可以在平台之間複製數據，共享文件和資料夾，還可以選擇以全螢幕模式執行 Windows。

Parallels 可以運行從 Windows 3.1 到目前 XP 版本的 Windows 系統。您需要自行提供 Windows 安裝軟體。Parallels 的一個缺點是它不支援 3D 加速圖形，這意味著一些高階 3D 遊戲和其他程式運行緩慢或不流暢。

**在執行 Parallels 的 Mac 筆記型電腦上按滑鼠右鍵，並在 Parallels 中啟用 Shift Key Ortho 切換**

Mac 筆記型電腦沒有滑鼠右鍵。 Rhino 卻很擅長使用滑鼠右鍵。如果您在 Mac 筆記型電腦上使用 Rhino，不妨為自己買一個無線兩鍵或三鍵滑鼠。

如果您使用 Parallels，它有個「偏好設定」，可以幫助使用者模擬滑鼠右鍵。如果您按住 Control 鍵並點擊（唯一的）(單鍵) 滑鼠按鈕，Parallels 會將其轉換為滑鼠右鍵。但這會干擾擾 Rhino 按住 Control 鍵並點擊取消選取物件的功能。

您可以在 Parallels 中關閉此轉換來還原 Control-click 功能。
開啟 Parallels 的“偏好設定”，然後選擇“鍵盤”標籤。
在**按滑鼠右鍵:** 的設定中，同時選擇**Ctrl** 和 **Command**，然後點選**OK**。
這將使模擬右鍵單擊變得難以執行，並實際上停用了 Parallels 的模擬右鍵行為。現在，您可以在 Rhino 中毫無問題地按住 Control 再點擊，甚至按住 Shift +Control 鍵再單擊。
如果您關閉了模擬右鍵單擊行為，最好使用具有真正滑鼠右鍵的滑鼠！
