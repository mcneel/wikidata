====== Questions souvent posées sur Rhino pour Mac ====== 

--> Puis-je acheter Rhino pour Mac?#

Non, Rhino pour Mac est actuellement en cours de développement. Il n'est pas en vente pour le moment.  [[http://www.rhino3d.com/download/wenatchee/5.0/wip|Téléchargez simplement Rhino pour Mac]] et profitez-en pendant la période de développement.  Tous vos commentaires seront les bienvenus. 

<--

-->Quand sortira Rhino pour Mac ?#

Nous ne le savons pas. McNeel & Associates a un cycle de développement inhabituel et des versions incomplètes de notre logiciel sont mises à disposition des utilisateurs afin d'obtenir leur avis. Rhino pour Mac suit le même style de développement que celui utilisé pour le [[http://www.rhino3d.com/nr.htm|développement de Rhino pour Windows]].

Pour obtenir la liste complète des erreurs restantes avant la sortie du produit, consultez [[http://mcneel.myjetbrains.com/youtrack/issues?q=project:MR+release+target:5.0+%23unresolved+sort+by:+State+|cette liste]].

<--
-->Lors de sa sortie, combien coûtera-t-il ?#

Nous n'avons pas encore déterminé notre politique de tarification mais il sera moins cher que Rhino pour Windows. Un tarif promotionnel avant la sortie sera disponible pour les utilisateurs possédant une licence de Rhino 5 pour Windows puis un tarif promotionnel sera proposé après la sortie à tous les utilisateurs pendant une durée limitée. 

<--

-->Si j'achète Rhino pour Windows, est-ce que j'aurai Rhino pour Mac gratuitement ? Quelle sera la politique de mise à niveau entre plate formes ?#

Les détenteurs d'une licence de Rhino 5 pour Windows pourront profiter d'une promotion pré-vente, pendant laquelle chaque licence de Rhino 5 pour Windows pourra être utilisée pour obtenir un tarif préférentiel sur une licence de Rhino pour Mac. 
<--

-->Qu'est-ce qu'une WIP ?#

WIP signifie Work In Progress en anglais, soit En cours de développement. C'est notre forme de "Bêta" et elle est souvent sortie en parallèle avec une version commerciale stable. La WIP contient des fonctions expérimentales et à moitié terminées. Les WIP sont normalement mises à disposition de tous les utilisateurs possédant une licence commerciale ou éducation du logiciel. La WIP de Rhino pour Mac est en cours de développement depuis de nombreuses années. Nous possédons actuellement une WIP préliminaire et nous publierons ensuite une autre WIP après la sortie du produit, qui sera mise à disposition de tous les utilisateurs ayant acheté une version complète. 
<--

--> Rhino pour Mac est limité dans le temps. Que se passe-t-il quand la version expire ?#

Cette expiration permet de s'assurer que tous les utilisateurs testent la dernière version et qui ne communiquent pas des problèmes qui ont déjà été corrigés. 

Avant que chaque version WIP de Rhino pour Mac n'expire, une nouvelle WIP sortira et pourra être [[http://www.rhino3d.com/download/wenatchee/5.0/wip|téléchargéee]]. Rhino pour Mac recherche les mises à jour à chaque démarrage et vous avertit lorsqu'une nouvelle version est disponible. Un message d'information est également publié dans la [[http://discourse.mcneel.com/category/rhino-for-mac|section de Rhino pour Mac]] du [[http://discourse.mcneel.com/ |forum de discussion]] chaque fois qu'une nouvelle version WIP est disponible.

<-- 

-->Après la sortie commerciale de Rhino pour Mac qu'en sera-t-il de la WIP ?#
La WIP préliminaire expirera quelques semaines après la sortie de la première version commerciale.  Une version WIP, mise à disposition uniquement des utilisateurs possédant une licence de Rhino pour Mac sera publiée après la sortie de la version commerciale. 
<--

-->Après la sortie de la version commercial de Rhino pour Mac, y aura-t-il des versions révisées ou mises à jour ?#
Oui.  Les mises à jour de Rhino pour Mac contiendront des corrections d'erreurs ainsi que de nouvelles fonctions. Par exemple, les mises en page et les sessions, qui ne seront pas incluses dans la première version, sont prévues dans les mises à jour ultérieures. Ceux qui ont acheté Rhino pour Mac recevront ces mises à jour gratuitement. 
<--

-->En quoi Rhino pour Mac est-il différent de Rhino pour Windows ?#
  * Rhino pour Mac permettra de réaliser presque toutes les opérations de modélisation que Rhino pour Windows peut faire.
  * Rhino pour Mac possède un moteur de rendu intégré, des objets d'annotation tels que les cotes, le texte et un gestionnaire d'impression simple.

__Contrairement à Rhino pour Windows, Rhino pour Mac ne possède pas__

  * La possibilité d'être enrichi par des modules ou des applicatifs, donc :
  * Il ne prend pas en charge RhinoCAM, RhinoGold, Matrix, Grasshopper.
  * De mises en page, sessions ni outils d'animation (mais ces derniers seront ajoutés lors de mises à jour).

...L'interface de Rhino pour Mac est relativement différente de celle de Rhino pour Windows.
<--

-->Est-ce que Rhino pour Mac fonctionne dans d'autres langues que l'anglais ?#

Oui. Elle est actuellement disponible en espagnol, français, allemand, italien, chinois  japonais et coréen. Cette version est toujours en cours de développement et la localisation n'est pas encore terminée. 

Rhino pour Macintosh utilise la langue par défaut de l'ordinateur. Pour utiliser une autre langue que celle du système d'exploitation :
  - Dans le menu, cliquez sur **Rhinoceros** puis sur **Preferences**.
  - Dans le panneau Preference, cliquez sur **Language** puis sélectionnez votre langue.

<-- 

-->Qu'est-ce qui ne fonctionne pas ? Pourquoi j'obtiens le message "Unknown or unimplemented command" ?#

Nous prévoyons d'introduire la plupart du programme de Rhino pour Windows sur la plate-forme Macintosh mais toutes les fonctions ne sont pas encore implantées dans cette version WIP de Rhino pour Mac. Consultez le [[http://www.rhino3d.com/nr.htm|processus de développement de logiciel]] appliqué à Rhino sur Windows pour voir comment nous travaillons sur les nouveaux produits. Nous suivons le même plan pour le développement de Rhino pour Mac. Nous sommes actuellement à la [[http://www.rhino3d.com/nr.htm#phase1|phase 1 : en cours de développement]]. 

Cette [[rhino:mac:whatworks|page Internet]] décrit ce qui fonctionne et ce qui doit encore être implanté.

<-- 

-->Comment puis-je désinstaller Rhino ?#

Les étapes suivantes suppriment complètement Rhino pour Mac de votre ordinateur :

  - Supprimez le programme dans le dossier Applications.
  - Supprimez le répertoire **/Library/Application Support/McNeel**. Ce répertoire contient les modèles des tutoriels et les manuels si vous les avez téléchargés, sinon il se peut qu'il n'existe pas sur votre ordinateur.
  - Supprimez le répertoire **~/Library/Application Support/McNeel** dans votre répertoire source. Ce répertoire n'est pas toujours utilisé par Rhino et il se peut qu'il n'existe pas sur votre ordinateur. Il peut contenir des matériaux de Toucan que vous avez créés, des scripts de Python et d'autres modules de Rhino pour Mac
  - Supprimez le fichier **~/Library/Preferences/com.mcneel.rhinoceros.plist** de votre répertoire source. Ce fichier contient toutes les préférences de Rhino.

==== Compatibilité ====

-->Les fichiers de données sont-ils compatibles avec la version Windows ?#

Oui, les fichiers de Rhino 5 sont identiques. Vous pouvez également exporter des fichiers vers la version 4 de Rhino pour Windows.

<-- 

-->Puis-je utiliser ma souris à deux ou trois boutons ?#

Oui, tous les ordinateurs Macintosh sont compatibles avec les souris à plusieurs boutons et à molette. Mais vous pouvez toujours utiliser une souris Macintosh à un seul bouton. Pour simuler l'utilisation du bouton de droite, maintenez la touche Ctrl tout en cliquant.

<-- 

-->Puis-je utiliser ma souris Spaceball/SpaceNavigator/SpaceExplorer ?#

Seul le SpaceNavigator peut être utilisé pour le moment.

<--

==== Scripts et modules ====

-->Puis-je lancer un script de commande sur la version Macintosh ?#

Oui. Pour charger et lancer un script de commandes, dans le menu Rhino, sélectionnez **Outils > Commandes > Lire dans un fichier**.
 
<--

-->Puis-je lancer un programme RhinoScript sur la version Macintosh ?#

Non. RhinoScript est basé sur VisualBasic et VisualBasic ne fonctionne pas sur OS X. Nous utilisons Python pour les scripts sur la version Mac et la version Windows et il sera ainsi possible d'écrire des scripts valables pour les deux versions de Rhino. Voir le [[http://discourse.mcneel.com/category/scripting | forum d'assistance pour l'écriture de scripts dans Rhino]] pour plus d'informations sur Python pour Windows et OS X.

<--


-->Les modules de tiers fonctionneront-ils sur Rhino pour Mac ?#

Peut-être. Cependant, les modules de Windows ne fonctionneront pas sur Mac. La décision de transférer un module vers Rhino pour Mac sera prise par chaque développeur de module.
<--
 
-->La version Macintosh sera-t-elle compatible avec mon module C ?#

Probablement dans quelques temps. Nous ne sommes pas encore prêt pour fournir une assistance sur les modules C++ mais nous espérons le faire dans le futur. Vous devrez recompiler votre module en utilisant le système de développement de Macintosh. Suivant la complexité de votre module, cela peut être assez facile de créer un module qui se compile et fonctionne aussi bien sur Windows que Macintosh.

<--

-->La version Macintosh sera-t-elle compatible avec mon module .NET ?#

Probablement dans quelques temps. Nous travaillons sur des outils qui permettront aux développeurs de module de Rhino d'écrire des modules qui fonctionneront aussi bien sur OS X que sur Windows.

==== Problèmes d'installation ====

-->Dois-je désinstaller Rhino avant d'installer une version plus récente ?#

Non. 

<-- 

-->OS X indique "Impossible d’ouvrir les images disques suivantes." Que dois-je faire ?#

Le fichier .dmg que vous avez téléchargé est probablement endommagé.  Vérifiez que votre fichier .dmg est valide. Les fichiers peuvent parfois être corrompus pendant le téléchargement.

  - Lancez l'utilitaire de disque **/Applications/Utilitaires/Utilitaire de disque**
  - Dans le menu, sélectionnez **Vérifier...** Une boîte de dialogue apparaît.
  - Sélectionnez le fichier .dmg que vous avez téléchargé. Il se trouve normalement dans le dossier Téléchargements et s'appelle rhinoceros_wip_YYYYMMDD.dmg
  - Si le fichier que vous avez téléchargé n'est pas valide, retournez sur la page de [[http://www.rhino3d.com/download/wenatchee/5.0/wip|téléchargement]] et demandez un autre lien de téléchargement.
  Si ce problème se reproduit, il se peut que votre connexion Internet soit instable, essayez de télécharger le fichier depuis une autre connexion Internet.  Le fichier de téléchargement fait plus de 100 Mo.

<-- 

-->J'ai installé la dernière version de Rhino et elle me dit encore qu'elle a expiré. Et maintenant ?#

Plusieurs copies de Rhino sont installées et celle qui démarre est la version expirée. Vous devez supprimer **toutes** les copies de Rhino. 

Double cliquez sur un fichier 3DM. Si le fichier s'ouvre dans Rhino, une copie de Rhino est toujours installée quelque part. 
  - Cliquez avec le bouton de droite sur l'icône de Rhino dans le Dock, sélectionnez **Options > Afficher dans le Finder** afin de trouver cette copie.
  - Déplacez cette copie dans la Corbeille. 
  - Lorsqu'il n'existe plus de copies de Rhino, double cliquer sur un fichier 3DM l'ouvrira dans TextEdit. Lorsque ceci se produit, toutes les copies de Rhinoceros ont été supprimées.
  - Réinstallez maintenant la version que vous venez de télécharger.

-->Une boîte de dialogue est apparue en indiquant "Program Failure" et Rhino pour Mac ne marche plus.#

Lorsque Rhino pour Mac rencontre une situation inattendue, il affiche une boîte de dialogue contenant le message "Program Failure". Après cette erreur, Rhino se ferme internement et vous ne pouvez plus lancer de commande ni utiliser le menu. Vous pouvez enregistrer votre fichier et vous pouvez fermer Rhino mais vous ne pourrez rien faire de plus. Nous avons besoin de savoir lorsque ceci se produit. 
Lorsque vous travaillez dans Rhino, vous pouvez voir la boîte de dialogue suivante :

Rhino affiche cette boîte de dialogue générique dans plusieurs cas. Toutes les fonctions ne sont pas encore implantées dans Rhino et vous verrez cette boîte de dialogue lorsque vous essaierez d'utiliser une fonction non implantée. Rhino possède également un code de programme qui ne devrait normalement être utilisé que dans la version Windows de Rhino. Si une des actions lancées essaye d'exécuter ce code, vous obtiendrez aussi un message d'erreur de Rhino. De nombreux tests de débogage sont actuellement réalisés sur ce code et vous verrez cette boîte de dialogue lorsqu'un test interne découvre une situation qui ne devrait pas se produire.

{{:rhino:mac:notimpl.png?nolink|}}

Lorsque vous rencontrez une de ces boîtes de dialogue, écrivez le nom du fichier, le numéro de la ligne et les informations sur la version contenue dans le message. Envoyez ces informations par mail, avec une description de ce que vous faisiez lorsque cette boîte de dialogue est apparue, à <mactech@mcneel.com>. Nous essaierons de reproduire le problème et de trouver une solution afin que cette erreur ne se produise plus dans le futur.

Lorsque cette boîte de dialogue est apparue Rhino ne permet de lancer que quelques commandes. Vous pourrez enregistrer votre fichier et quitter Rhino mais peu d'autres choses fonctionneront.

--> J'aimerai cliquer avec le bouton de droite pour répéter la dernière commande #

La désactivation du menu contextuel permet d'activer la répétition de la dernière commande.

  - Dans le menu, cliquez sur **Rhinoceros** puis sur **Preferences**.
  - Sélectionnez le panneau **Mouse**.
  - Dans la section **Bouton droit de la souris**, désactivez la case **Enable Context Menus**.
