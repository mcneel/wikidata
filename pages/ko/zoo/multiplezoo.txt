======Zoo를 실행하면 강제로 서비스가 종료되는 경우======
> **Product:** //[[ko:zoo:home|Zoo]]//
> **Summary:** //여러 개의 Zoo 서버 사용 방법//

=====질문=====
Zoo 라이센스 관리자를 시작하면 바로 종료됩니다. 예전에는 제대로 실행되었습니다. Zoo 소프트웨어를 재설치하니 Zoo 실행 후 바로 종료되는 현상이 나타났습니다. 낮 동안에는 서버를 끌 수 없기 때문에, 아직 서버를 리부트해보진 못 했습니다. 어떻게 하면 해결할 수 있는지 알려 주세요. 감사합니다.

=====답변=====
Zoo 로그 파일 (Zoo 콘솔에서 **Tools**를 선택한 후, **View Event Log**를 선택)을 보면 사용하시는 네트워크상에 또 다른 Zoo 서버가 있는 것을 확인할 수 있을 것입니다.

네트워크에서 두 개 이상의 Zoo 서비스를 허용하려면,  
Zoo 콘솔에서 **Tools**를 선택한 후, **Options**를 선택합니다. **Allow multiple Zoo servers to run on this network(이 네트워크에서 여러 개의 Zoo 서버가 실행되도록 허용)** 옵션의 확인란을 선택합니다. Zoo 서비스를 다시 시작합니다.

=====추가 정보=====
과거에는 광역 커뮤니케이션만이 지원되어, 한 도메인 또는 작업그룹에서 하나의 Zoo만을 실행시킬 수 있었습니다.  그러나, 이제는 라우트된 네트워크도 Zoo에서 지원하므로, 두 개 이상의 서버를 실행해야만 하는 경우가 있을 수 있습니다. **Allow multiple Zoo servers to run on this network(이 네트워크에서 여러 개의 Zoo 서버가 실행되도록 허용)** 기능을 허용하면 가능합니다.

이 기능을 사용으로 설정하면, Rhino 클라이언트에서 반드시 DNS 또는 Registry Zoo 조회가 지원되어야 합니다. 

Zoo 도움말의 **Installing on Routed Networks (라우트된 네트워크에서의 설치 방법)**을 참조하시기 바랍니다.
