=====Problemas del instalador de Rhino 5=====
El instalador de Rhino 5 mostrará un archivo de registro cuando se produzca un error en el proceso de instalación. Este archivo de registro ayudará al equipo de soporte técnico a resolver los problemas que tenga. 
  - Cuando aparece el mensaje de error, hay un enlace en el diálogo. 
  - Haga clic en el enlace para abrir el registro y seleccione "Guardar como..." archivo *.txt.
  - Envíe un e-mail adjuntando el archivo de texto a [[tech.eu@mcneel.com]]
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]
=====Error 1603 del instalador de Rhino 5 y otros=====
El **código de error 1603** no es un error de bloqueo específico, es un error general en proceso de instalación. Algunas de las condiciones que pueden causar el error 1603 son:
* La creación de nombres de archivos cortos está desactivada en el equipo de destino.
  * Una acción personalizada del script de instalación se ha prototipado incorrectamente.
  * Un archivo se ha bloqueado y no puede sobrescribirse.
  * El servicio Windows Installer no se ha instalado correctamente.
  * Las carpetas temporales de Windows están llenas.
  * El instalador se dañó después de la instalación y, por tanto, aparece este error durante la desinstalación.
Si el instalador de Rhino se bloquea y aparece el error 1603 o no aparece ningún código de error, puede probar los pasos siguientes para obtener más información.

**//Paso 1//**
  - Siga los pasos indicados en Errores del instalador de Rhino 5 para enviarnos el Registro de instalación. 
  - Cierre el instalador existente
  - Abra el Explorador de Windows y vaya al contenido del DVD de instalación de Rhino
  - Abra la carpeta Installer del DVD
  - Abra la carpeta Prerequisites.
  - Ejecute el instalador [[http://www.microsoft.com/en-us/download/details.aspx?id=17718|dotNetFx40_Full_x86_x64.exe]] que se incluye en la carpeta.
  - Si el instalador dotNet muestra un error, tome nota del error y continúe con el Paso 2.
  - Si el instalador dotNet funciona correctamente, continúe con el Paso 3.

**//Paso 2//**
  - Haga clic aquí para ejecutar el [[http://windows.microsoft.com/es-es/windows/help/windows-update|servicio de Windows Update]]
  - Si hay actualizaciones importantes para instalar, haga clic en Actualizar ahora para instalar las actualizaciones. Este proceso puede tardar bastante.
  - Si se produce un error al ejecutar el instalador de Windows Update, consulte este artículo [[http://support.microsoft.com/kb/906602/es|Solucionar problemas de instalación habituales en Windows Update]]
  - Si resuelve el problema con Windows Update y puede ejecutar el [[http://windows.microsoft.com/es-es/windows/help/windows-update|servicio Windows Update]], vuelva al Paso 1.

**//Paso 3//**
  - Vaya al DVD del instalador de Rhino a través del Explorador de Windows
  - Intente ejecutar el archivo Setup.exe en el directorio raíz del DVD.
  - La instalación debería completarse.
  - Si el instalador continúa fallando, envíe el archivo de registro adjunto en un e-mail a tech.eu@mcneel.com 
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]

=====No se pudo abrir la clave: DESCONOCIDO\Componentes (Error 1402)=====
** //Pregunta://** //Mensaje "No se pudo abrir la clave: DESCONOCIDO\Componentes\(letras y números). Error 1402. Compruebe que dispone de los derechos de acceso necesarios para esa clave o póngase en contacto con el personal de soporte."//

**//Causa://** Una o más claves del registro de la clave HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Installer\UserData\S-1-5-<numbers>\Components tiene permisos incorrectos.

**//Solución 1://**
El el menú Inicio de Windows, escriba cmd en la casilla de búsqueda y pulse Intro. Copie la línea siguiente, péguela en la ventana de cmd y pulse Intro. Entonces Rhino podrá instalarse.

<code>
secedit /configure /cfg %windir%\inf\defltbase.inf /db defltbase.sdb /verbose
</code>

**//Solución 2://**
//AVISO: El siguiente procedimiento implica la edición del Registro de Windows. La edición incorrecta del registro puede dañar el equipo y evitar que se inicie. Si no se siente seguro para realizar los siguiente pasos, póngase en contacto con el soporte técnico de McNeel.//
  - Descargue [[http://technet.microsoft.com/en-us/sysinternals/bb897553|psexec]] de Microsoft TechNet y extraiga el contenido en el escritorio.
  - Abra un shell de comandos de Windows
  - Cambie los directorios en el escritorio.
  - En el shell de comandos, escriba: **psexec -i -d -s c:\windows\regedit.exe** Windows ejecutará el editor del registro con los permisos de la cuenta del sistema, una cuenta más potente que la de administrador.
  - Vaya a a clave HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Installer\UserData\
  - Haga clic con el botón derecho en la carpeta **UserData** y luego en **Permisos...**
  - En la opción Permisos del cuadro de diálogo UserData, haga clic en **Avanzado**
  - Asegúrese de que "Administradores" tiene "Control total" de los permisos de esta carpeta y de que "Usuarios" tiene permisos de "Lectura". Habrá otros permisos de usuarios, pero el de Administradores y Usuarios es el más importante.
  - Seleccione el **Reemplazar los permisos de todos los objeto descendientes con permisos heredables de este objeto**.
  - Haga clic en **Aceptar** para aplicar los permisos.
  - Vuelva a instalar Rhino.
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]

=====Instalación de la última Service Release en un nuevo equipo=====

**//Pregunta:://** //Tengo un nuevo equipo y quiero instalar la versión de Rhino **X** SR **Y**.  Siempre he comprado versiones de actualización de Rhino.  ¿Tengo que instalar TODAS las versiones anteriores y los ''service packs'' sucesivos??//

**//Respuesta://** Simplemente instale la última actualización que tiene en el CD.  Durante la instalación, se le solicitará insertar el CD de una versión anterior para validar que lo tiene, pero no es necesario '''INSTALAR''' las versiones anteriores.  Cuando haya instalado la última versión del CD, continúe con la instalación de la última Service Release (no es necesario instalar las anteriores).
\\
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]
=====¿Qué es una Service Release?=====
Las versiones Service Release no tienen ningún coste, son parches para las instalaciones existentes del programa que incluyen correcciones de errores y actualizaciones. Estos instaladores se pueden descargar y sólo se instalarán si tiene una versión del producto ya instalada.  Las Service Release se identifican de dos maneras: el número de SR, como SR3 o SR4b, y la fecha, que aparece en el nombre del archivo de descarga, por ejemplo 20050729, que significa 29 de julio de 2005.
\\
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]
=====¿Cómo sé qué versión está instalada?=====
Una vez el software está instalado, puede ver la fecha y el número de SR visualizando la información "Acerca de", que aparece en la pantalla de presentación al iniciar Rhino o yendo a Rhino > Ayuda > Acerca de Rhinoceros [[rhino:rhinoceros|Rhinoceros]].
\\
Para Flamingo, ejecute el comando FlamingoAcercaDe. Para Bongo, ejecute el comando BongoAcercaDe.
\\
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]
=====¿Cómo puedo obtener una Service Release?=====
El modo más rápido es usar el servicio de Buscar actualizaciones.  En Rhino, vaya al menú Ayuda y haga clic en Buscar actualizaciones.  Si está conectado a Internet, los servidores web de McNeel le ofrecerán la Service Release correcta si hay una más nueva que la versión que tiene instalada.  Para Flamingo, vaya al menú Trazado de rayos o Fotométrico y haga clic en Buscar actualizaciones.  Para Bongo, vaya al menú Bongo y haga clic en Buscar actualizaciones.
\\
En Rhino 5, vaya a **Herramientas, Opciones, Actualizaciones y Service Releases** y establezca la frecuencia de actualización a "Service Releases" o "Release Candidates". Cuando cierre Rhino, se le solicitará aprobar la instalación de la última versión. Puede rechazar la solicitud y ejecutarlo más adelante.
\\
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]
=====¿Puedo eliminar versiones anteriores desde "Añadir o quitar programas" sin causar problemas?=====
La opción "Añadir o quitar programas" del Panel de control de Windows contiene todas las Service Release instaladas, y cada una usa aprox. 250MB de espacio en el disco. Esto es un poco engañoso. Windows indica el total como si todos los archivos instalados se hubieran agregado a la instalación existente, aunque muchos archivos se hayan sobrescrito en la instalación. **'No**' utilice la opción de "Añadir o quitar programas" para quitar las Service Release de Rhino 4.0.
\\
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]

=====Error 1327 - Unidad no válida:=====
**//Pregunta://** //Me aparece el "Error 1327. Unidad no válida (E:, F:, etc)" durante la instalación//

**//Causa://** //Su cuenta de usuario tiene opciones que indican que una o más unidades no existentes debe utilizarse para carpetas especiales como "Mis documentos"//

**//Solución://**
  - En el menú Inicio, haga clic en Ejecutar.
  - Escriba RegEdit y pulse Ejecutar
  - Vaya a HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Shell Folders
  - Buque los directorios listados que contienen la letra de la unidad mencionada en el error del instalador. Cámbielos de manera que abran una unidad y carpeta válidas.
  - Vaya a HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders
  - Buque los directorios listados que contienen la letra de la unidad mencionada en el error 

**Nota:** Este método o tarea contiene pasos que modifican el registro. Pueden producirse problemas graves si modifica incorrectamente el registro. Por tanto, asegúrese seguir bien estos pasos. Para mayor protección, realice una copia de seguridad del registro antes de modificarlo. Podrá restaurar el registro si se produce un problema. Para obtener más información sobre cómo realizar una copia de seguridad y restaurar el registro, haga clic en el número de artículo siguiente para verlo en el Microsoft Knowledge Base:
[[http://support.microsoft.com/kb/322756|Cómo realizar una copia de seguridad y restaurar el registro en Windows ]]
\\
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]
=====Error 1935=====
**//Pregunta://** //Me aparece el "Error 1935. Ha ocurrido un error durante la instalación del ensamblaje "Microsoft.VC80.CRT, ..."//

**//Causa://** //Se trata de un error del Microsoft C++ 2005 Redistributable Package//

**//Solución 1://**
  * **Desactive el antivirus en el sistema.** Un usuario indicó que la desactivación de AVG permitía la instalación de Rhino.
**//Solución 2://**
  * **Intente instalar varias veces.** Un usuario informó que la instalación de Rhino función la tercera vez que intentó instalar el programa. No hizo nada en el sistema.
**//Solución 3://**
  - En Windows Vista o Windows 7, en el menú Inicio, escriba Características.
  - Haga clic en **Activar o desactivar las características de Windows**
  - En el cuadro de diálogo Características de Windows, seleccione la casilla **Activación HTTP de Windows Communication Foundation**:
  - {{:rhino:rhino_1935_windows_features.png|}}
  - Haga clic en Aceptar

Si no puede activar esta característica o si el cuadro de diálogo Características de Windows aparece vacío, es porque la instalación de Windows está dañada y debe repararse. Póngase en contacto con el soporte técnico de Microsoft para obtener ayuda.

**//Más información://**
Las personas que experimentan este error al instalar Rhino, tampoco pueden instalar el [[http://www.microsoft.com/downloads/en/details.aspx?FamilyID=32bc1bee-a3f9-4c13-9c99-220b62a191ee&displaylang=en|Microsoft C++ 2005 Redistributable Package]] directamente de Microsoft. Por tanto, hay un problema con Windows y Microsoft Redistributable Package, no con el instalador de Rhino.
\\
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]
=====Error 2502=====
**//Pregunta://** Cuando Rhino 5 informa de que hay una actualización nueva para instalar, elijo instalar la actualización y aparece el "Error 2502".

**{{http://wiki.mcneel.com/_media/rhino/k01373.png?400}}**

**//Solución://**

=====Error del instalador .NET Framework 4 (HRESULT: 0xc8000222)=====
**Problema:**\\ .NET Framework 4 no se pudo instalar debido al error HRESULT: 0xc8000222.

**Causa:**\\ Algunos archivos de la carpeta **Software Distribution** están dañados.

**Solución:**
  - Descargue [[http://files.na.mcneel.com/misc/Fix_0xc8000222.zip|Fix_0xc8000222.zip]].
  - Extraiga el archivo **Fix_0xc8000222.bat** del zip en una carpeta.
- Haga clic con el botón derecho en Fix_0xc8000222.bat y seleccione **Ejecutar como administrador**.\\ <color #999999>Lo que hace este archivo:\\ 1) Desactiva </color><color black>Windows Update Service</color><color #999999>.\\ 2) Elimina </color><color black>C:\Windows\SoftwareDistribution</color><color #999999>.\\ 3) Vuelve a activar </color><color black>Windows Update Service</color><color #999999>.</color>
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]
=====El instalador de Rhino 5 solicita .NET Framework 4=====
**Problema:**\\El instalador de Rhino 5 solicitecita la instalación de .NET Framework 4 en primer lugar y no permite instalar Rhino 5, pero .NET Framework 4 ya está instalado en el sistema.

**Causa:**\\ .NET Framework 4 está bloqueado en el sistema.

**Solución:**
  - Descargue <color #999999>[[http://files.na.mcneel.com/misc/dotnetfx_cleanup_tool.zip|.NET Framework Cleanup tool]]</color>.
  - Ejecute .NET Framework Cleanup tool.
  - Seleccione el producto **.NET Framework - All versions** para la limpieza.\\ {{http://wiki.mcneel.com/_media/rhino/rhino5/k01521.png?w=405}}
  - Haga clic en el botón **Cleanup Now**.
  - Reinicie el ordenador.
  - Instale Rhino 5 desde el DVD de Rhino 5.\\ <color #999999>El instalador del DVD de Rhino 5 instalará .NET Framework 4 en primer lugar y luego instalará Rhino 5.\\ Si está instalando la </color><color black>versión de evaluación de Rhino 5</color><color #999999>, necesitará [[http://www.microsoft.com/en-us/download/details.aspx?id=17718|descargar]] y reinstalar la versión completa de .NET Framework 4 antes de instalar Rhino 5. </color>
[[http://wiki.mcneel.com/es/rhino/installproblems|Volver arriba.]]

===== ¿Windows 7/8 es un sistema compatible? =====
**Problema:**\\ Estoy utilizando Windows 7/8, per el instalador del DVD de Rhino 5 dice que mi sistema no es compatible.

**Solución:**\\ 
  - Haga clic con el botón derecho en el instalador exe y seleccione **Propiedades**.
  - Vaya a la ficha **Compatibilidad**.\\ <color #999999>El modo de compatibilidad puede estar definido en "Windows XP (Service Pack 2)" sin poder modificarse.</color>
  - Haga clic en **Cambiar la configuración para todos los usuarios** cerca de la parte inferior.
  - Desactive la casilla **Ejecutar este programa en modo de compatibilidad**. 
  - Haga clic en **Aceptar** dos veces para cerrar los diálogos.
  - Vuelva a hacer clic con el botón derecho en el instalador exe y seleccione "Ejecutar como administrador" para instalar.
