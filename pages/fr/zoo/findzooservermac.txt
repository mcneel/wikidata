======Comment Rhino pour Mac trouve-t-il le serveur Zoo LAN ?======

>**Produit :** //[[zoo:home|Zoo LAN]]//
> **Résumé :** //Cet article offre un aperçu de la manière dont Rhino pour Mac trouve un serveur de gestionnaire de licences Zoo LAN.//

Lorsque Rhino doit extraire une licence sur un serveur de Zoo LAN, il détermine le nom du serveur de Zoo LAN en regardant dans les destinations suivantes, dans cet ordre :
  - Le fichier xml de paramètres ZooClient. Il recherche le nom de l’hôte ou l’adresse IP de votre serveur Zoo LAN.
  - Il envoie une requête à votre serveur DNS (système de nom de domaine) pour demander le nom du serveur de Zoo LAN par défaut.


=====Plus d'informations=====

====Fichier de paramètres ZooClient====

Lorsqu’il est exécuté en tant que nœud de réseau, Rhino détermine le nom du serveur de Zoo LAN en regardant d’abord dans le fichier **LicensesZooClient.Settings.xml** dans ce dossier :

===Rhino 8===

<code>
/Users/Shared/McNeel/Rhinoceros/8.0/License Manager
</code>

===Rhino 7===

<code>
/Users/Shared/McNeel/Rhinoceros/7.0/License Manager
</code>

===Rhino 6===

<code>
/Users/Shared/McNeel/Rhinoceros/6.0/License Manager
</code>


Le contenu du fichier peut être visualisé et modifié dans un éditeur de texte. Le contenu du fichier est similaire à l’exemple suivant :

<code>
<?xml version="1.0" encoding="utf-8"?>
<!--Zoo client settings file written by Rhino-->
<ZooClient>
  <Software_McNeel_Rhinoceros_6.0_License_Manager Server="host name or IP address" />
</ZooClient>
</code>

Où : **"host name or IP address"** est le nom de l’hôte ou de l’adresse IP de votre serveur Zoo LAN.

>**Remarque :** //Pour que Rhino puisse créer ce fichier, l’utilisateur qui exécute Rhino doit avoir l’autorisation nécessaire pour créer des fichiers à cet emplacement.

> **De plus,** //L’étiquette xml **Software_McNeel_Rhinoceros_6.0_License_Manager** figurant ci-dessus est utilisée par **toutes** les versions de Rhino, //pas seulement Rhino 6//.//

====Utilisation du DNS===
Si Rhino ne peut pas déterminer le nom du serveur Zoo en cherchant dans le fichier de paramètres ZooClient, il enverra une requête à votre serveur DNS concernant le serveur du Zoo par défaut, qui possède le nom d'hôte suivant :

<code>
__mcneel.__zoo5
</code>

Par exemple, si le domaine de votre entreprise est //entrep.com//, vous pouvez ajouter un alias DNS de //<nowiki>__mcneel.__zoo5.entrep.com"</nowiki>// qui indique votre serveur de Zoo.
