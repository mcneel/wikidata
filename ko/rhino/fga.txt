====== 자주 드리는 대답 -- Rhino ======

==크래시 보고는 어떻게 보내나요?==
  * 크래시를 반복하실 수 있습니까? 반복이 가능하다면, 개발자들이 해당 크래시를 반복시키기 위해 필요한 과정을 상세하게 알아야 합니다. 반복이 가능한 문제여야만 소프트웨어에서 해당 문제를 찾아 수정이 가능합니다. 
  * Rhino에서 크래시가 발생하고 사용 가능한 메모리가 충분하다면 크래시 보고 시스템이 시작합니다. 이메일 주소를 입력하시고 Rhino가 크래시할 때 어떤 작업을 하려고 하셨는지 설명해 주세요. 크래시 발생 직전에 사용하신 명령들의 목록도 적어 주시면 도움이 됩니다.
  * 가능하시면, 3dm 파일도 보고에 첨부하여 보내 주세요.
  * 이메일 주소와 설명을 적어 주시면 **개발자가 직접 문제를 분석합니다**. 이메일 주소와 설명을 적지 않으시면 보내주신 크래시 보고는 자동 시스템을 통해 프로세스됩니다. 

----

==안전 모드==
  * Rhinoceros 프로그램 그룹에 바로 가기가 있습니다. 안전 모드는 OpenGL과 플러그인 로드를 막는 문제 해결 기능입니다. 5 버전에서는 도구모음과 RDK 로드도 블록됩니다. 일반적으로 로드 여부를 묻는 화면에서 "예"를 선택합니다. 안전 모드는 문제 해결을 위한 도구이며, 작업 용도가 아닙니다.

----
==5 플러그인 해제==
  -Rhino 메뉴에서 도구 - 옵션 - 플러그인으로 가서 Rhino와 함께 배송되지 않는 모든 플러그인의 "사용" 항목을 해제합니다.
  -Rhino를 정상적으로 다시 시작하고 크래시 문제가 더 이상 발생하지 않는지 확인합니다. 플러그인 중 하나를 다시 사용으로 설정하고 Rhino를 다시 시작하여 크래시가 반복되는지 확인합니다. 다시 크래시가 발생하지 않는다면 사용으로 설정한 해당 플러그인은 문제가 아닙니다. 문제를 발생시키는 플러그인을 찾을 때까지 확인 과정을 반복합니다.

----

==Rhino 5가 시작하지 않음==
  * 잘못된 디스플레이 드라이버 또는 잘못된 플러그인으로 인한 문제일 수 있습니다. 문제의 원인을 파악하는 방법은 다음과 같습니다. 안전 모드로 Rhino를 시작하고 (Rhinoceros 프로그램 그룹에 바로 가기가 있습니다), 도구 - 옵션 - 뷰 - OpenGL 에서 가속 하드웨어 모드 사용 옵션을 해제합니다. Rhino를 닫고 정상적으로 Rhino를 시작합니다. 크래시 발생 없이 Rhino가 시작한다면, 문제의 원인은 아마도 디스플레이 드라이버라고 생각됩니다. 업데이트를 찾아 설치하세요. 업데이트된 디스플레이 드라이버를 설치한 후에도 Rhino에서 크래시가 발생한다면 문제의 원인은 플러그인으로 보입니다.

  * 이 경우, 안전 모드로 다시 시작하고 가속 하드웨어 그래픽의 확인란을 다시 선택합니다. 도구 - 옵션 - 플러그인으로 다시 가서 Rhino와 함께 배송되지 않은 모든 플러그인의 사용 항목을 모두 해제하고 "사용 안 함으로 설정된 플러그인 로드할 때 확인" 항목을 선택합니다. 안전 모드가 아닌, 정상적으로 Rhino를 시작하고 로드 여부를 묻는 창에서 모두 "아니요"를 선택합니다. Rhino 가 시작된다면, 문제는 로드 보호된 플러그인 중 하나입니다. Rhino를 정상적으로 다시 시작하고 이번에는 첫 번째 플러그인 로드에 예를 누르고 나머지는 "아니요"를 선택합니다. 첫 번째 플러그인을 로드할 때 크래시가 발생한다면 해당 플러그인이 문제입니다. 크래시가 발행하지 않는다면 문제가 아닙니다.  플러그인 페이지로 되돌아가서 해당 플러그인을 사용으로 설정합니다. 다시 Rhino를 시작하고 크래시가 발생하는 플러그인을 찾을 때까지 과정을 반복합니다.

==제가 갖고 있는 두 컴퓨터에 Rhino를 모두 설치할 수 있습니까?==
  * 예. 라이선스 계약은, 사용자가 직접 관리하는 모든 컴퓨터에 Rhino를 설치할 수 있도록 허용합니다. 단, 한 번에 한 대의 컴퓨터에서 Rhino를 실행해야 합니다. 동시에 한 대보다 많은 컴퓨터에서 Rhino를 실행하실 수 없습니다. 친구나 지인에게 Rhino를 "빌려"줄 수 없습니다.

==Rhino 업그레이드된 버전 설치하기==
  * **//질문://** //"Rhino 5 **업그레이드**를 설치하려면 이전 버전에 대한 어떤 정보가 필요한가요?"// 
  * **//대답://** Rhino 5 업그레이드 설치 관리자는 다음 중 하나가 해당되면 설치됩니다.
  
  * 순서대로 확인됩니다:
<code>
    Rhino 4.0 이 설치되어 있습니다
    Rhino 3.0 이 설치되어 있습니다
    Rhino 2.0 이 설치되어 있습니다
    설치 과정 중에 Rhino 4.0 키 코드가 입력됩니다
    설치 과정 중에 Rhino 3.0 키 코드가 입력됩니다
    설치 과정 중에 Rhino 2.0 키 코드가 입력됩니다
</code>    
  * 이전 버전을 실제로 설치하실 필요는 없으며, 이전 버전의 CD-Key만 있으면 됩니다.

==파일을 열려고 했을 때 읽을 수 없다는 오류 메시지가 표시되었습니다==
  * 시스템이 실행되는 과정은 다음과 같이 2 부분으로 나뉩니다:
    - 자동 저장 - Rhino에서 자동 저장을 설정해두었다면 옵션에 지정된 폴더에 임시 파일이 작성되고 Rhino가 실행 중이라는 플래그가 설정됩니다. Rhino가 정상적으로 종료되면 자동 저장된 파일이 삭제되고 Rhino가 플래그를 지웁니다. Rhino가 크래시하면 다음에 Rhino가 시작할 때까지 플래그가 지워지지 않고 자동 저장된 파일이 계속 존재하게 됩니다. 이 경우, Rhino가 정상적으로 종료되지 않았다는 경고가 표시되고 자동 저장된 파일의 작업 내용을 저장할 수 있는 한 번의 기회를 드립니다. 아니요를 선택하면 해당 파일을 잃습니다.
    - 백업 - Rhino 4 버전 SR7 부터 자동 백업 기능이 추가되었습니다. 이 기능을 해제하지 않은 경우, 기존 파일을 열면 새로운 편집 세션의 변경 내역이 해당 파일의 새로운 복사본에 저장되고, 사용자가 열었던 기존 파일은 파일 이름이 연결되어 ".bak" 확장자 파일로 저장되고 원래 파일과 동일한 폴더에 남습니다. 
  * 5 버전은 *.3dmbak 파일을 만들고 파일 열기 대화상자에서 해당 유형이 목록에 포함되어 있습니다. 
  * 4 버전의 경우, 파일 탐색기에서 목록 또는 자세히 보기를 사용하여 알려진 파일 형식의 확장자를 표시하도록 설정하고 [파일이름].3dm.bak 파일을 [새파일이름].3dm으로 변경합니다. 다른 여느 Rhino 3dm 파일처럼 열 수 있습니다.

==Rhino를 설치했을 때 Zoo 사용 옵션을 선택했으나, 저는 Zoo를 사용하지 않습니다==
  * Windows Vista 또는 Windows 7에서 다음 디렉터리의 모든 파일을 삭제합니다: C:\ProgramData\McNeel\Rhinoceros\5.0\License Manager\Licenses 
  * Windows XP에서 다음 디렉터리의 모든 파일을 삭제합니다: C:\Documents and Settings\All Users\Application Data\McNeel\Rhinoceros\5.0\License Manager\Licenses
  - Rhino를 다시 시작합니다.
  - CD-Key를 입력하라는 메시지가 표시되면 CD-Key를 입력합니다.

안내: 일부 시스템의 경우, 경로에 "\All Users\" 폴더 대신, "\Domain\" 로그인 이름이 있을 수 있습니다.
