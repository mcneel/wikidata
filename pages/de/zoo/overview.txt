======Zoo - Funktionsübersicht======
> **Produkt:** //[[de:zoo:home|Zoo]]//
> **Zusammenfassung:** //Ein Überblick über die Funktionen des Zoos.//

=====Überblick=====
Mit dem Zoo können Lizenzen von Rhino, Flamingo, Penguin, Brazil und Bongo in der gleichen Netzwerk-Arbeitsgruppe gemeinsam genutzt werden. Wenn diese Produkte als Mehrplatzsysteme anstatt als Einzelplatzsysteme installiert werden, funktioniert es folgendermaßen: 

  * Wenn ein Mehrplatzsystem startet, wird eine Lizenz vom Zoo angefordert. 
  * Eine Lizenz, die nicht verwendet wird, wird diesem System zugewiesen. 
  * Wenn eine Sitzung beendet wird, wird die Lizenz erneut in den Speicher verfügbarer Lizenzen des Zoos zurückgeschickt. 


=====Features=====
  * **Der Zoo ist kostenlos!**
  * **Keine zusätzlichen Kosten.** Es werden keine Sonderversionen von Rhino, Flamingo, Penguin, Brazil oder Bongo benötigt. 
  * **Keine spezielle Hardware nötig.** Der Zoo läuft auf jedem Windows-Rechner. 
  * **Keine spezielle Software nötig.** Der Zoo läuft auf jedem Windows-System, das das Microsoft .NET Framework 4.5 unterstützt. Dazu gehören Windows Vista und neuere Systeme.
  * **Einfache Einrichtung.** Sie können den Zoo auf einem beliebigen Windows-System installieren und die Lizenzschlüssel zentral im Zoo statt auf jedem Computer einzeln eingeben. 
  * **Lizenzabmeldung** Lizenzen können abgemeldet werden, d.h. Laptop-Benutzer können das Netzwerk verlassen. Wenn der Laptop wieder mit dem Netzwerk verbunden wird, kann die Lizenz wieder angemeldet werden. 
  * **Robust.** Netzwerkknoten funktionieren weiter, auch wenn nicht auf den Server zugegriffen werden kann.
  * **Routfähig.** Der Zoo funktioniert auf Einzelsegment-Netzwerken und auf Netzwerken, bei denen Systeme durch Router getrennt sind. 
  * **Ausführung als Dienst.** Beim Neustart des Zoo-Servers wird der Zoo automatisch gestartet. 

=====Weitere Informationen=====
  * **Unterstützung des Standard-Internet-Protokolls.** Clients kommunizieren mit dem Zoo über das **HTTP-Protokoll (Port 80)**. Für Systemverwalter ist es daher einfacher, Zoo-Anforderungen in komplexen Netzwerken oder über das Internet weiterzuleiten.
  * **Firewall-freundlich.** Der Zoo erfordert eine regelmäßige Kommunikation des Clients mit dem Server, initialisiert selbst aber keine Verbindung zurück zum Client. Verwalter müssen keine Firewallports auf Client-Computern öffnen.
  * **Befristete Lizenz-Abmeldung** Die vorige Zoo-Version unterstützte zwar bereits das Abmelden von Lizenzen für Laptop-Anwender oder Geschäftsreisende, es konnte aber noch keine beschränkte Abmeldedauer festgelegt werden. Administrator können nun die Dauer der Lizenzabmeldung festlegen.
  * **Unterstützung für Plug-ins von Drittentwicklern** Der Zoo erlaubt es Drittentwicklern von Plug-ins, Zoo-Support für ihre Produkte hinzuzufügen.
