====== Nvidiaディスプレイドライバの問題と情報 ======

===== Rhinoceros 5のQuadroドライバのテスト =====
**下のドライバがRhinoとQuadroカードに「最適」のドライバとして推奨されます。**

  * Windows 7で最も新しく安定しているドライバとして推奨するのは、[[http://www.nvidia.com/content/DriverDownload-March2009/confirmation.php?url=/Windows/Quadro_Certified/320.92/320.92-quadro-tesla-grid-desktop-win8-win7-winvista-64bit-international-whql.exe&lang=us&type=Quadro|320.92]]です。
  * Windows 8と8.1には、推奨できる安定したドライバはありません。
  * 新しいドライバが次々とリリースされているため、この情報は予告なしに変更されることがあります。

この一年間、特定のNVidia Quadroビデオカードをお使いのユーザーの方達よりビデオドライバのクラッシュについてのいくつかのご報告がありました。Rhinoの開発チームがこの問題を回避するまたは修正できる方法を探し出そうとしましたが、非常に困難です。 

この問題は、ビデオドライバの中で起こっているのです。Rhinoの中で行っている何かがクラッシュの引き金になっているとしても、ドライバはクラッシュするべきではないのです。そのため、Rhino側でこの問題を修正するのは非常に難しく、今の時点では不可能だと言えます。

この問題は、クラッシュのために作業が失われることと、Rhinoが終了されるために、私共も非常に心配しています。クラッシュによっては、Rhinoを実行していなくても、ウェブブラウザを実行して動画を見ていたりするだけでシステムがクラッシュする場合もありました。しかし、Rhinoとブラウザの組み合わせでクラッシュが起きる頻度が増すようです。

より詳しい詳細については、[[http://s3.amazonaws.com/mcneel/rhino/5.0/WIKI-techsupport/Quadro Testing for Rhino5_JL.pdf|こちら]]（英語）をご覧ください。

===== Nvidiaカードを使っている場合のランダムなクラッシュ - Nvidia Threaded Optimization設定 =====

何人かのユーザーの方から更新したNvidiaのドライバでランダムなクラッシュが起きているとのご報告がありました。弊社のJeffがこの問題を引き起こしていると考えられるNvidiaのコントロールパネルのある設定を発見しました。Nvidiaは、その設定は通常オフになっていると言いますが、それが正しいとは限りません。

<color coral>**このThreaded Optimizationコントロールをオフにすることを推奨します。**</color>.\\

> <color slategray>**Update:** //on 7-Nov-08, John (<jrharvey5923@gmail.com>) reported on the Rhino Newsgroup that there are no longer problems with threaded optimization in the newest drivers (ver. 180.42). Also these drivers reportedly allow multi-monitors on SLI computers.//</color>

> <color slategray>**Update:** //on 18-Nov-08, John Brock (<jb@mcneel.com>) is running a [[rhino:geforce|GeForce]] 9800 GTX with driver version 7.15.11.7813 (9/17/2008) and has the random crash problem with threaded optimization set to "auto".  Set to off eliminates the problem.//</color>
