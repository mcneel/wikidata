======Cloud Zoo 라이선스 사용 방식 설치 및 관리======

Cloud Zoo 라이선스 사용 방식은 Rhino 6이 라이선스를 사용하는 가장 새로운 방식입니다. [[ko:rhino:6:license|Rhino 6 라이선스 사용 방식]]의 개요를 참조하세요.

=====Cloud Zoo 개인용 라이선스 설정=====

  - [[https://accounts.rhino3d.com/help#create_account|Rhino 계정을 만듭니다]].
  - [[https://accounts.rhino3d.com/help#licenses|라이선스를 나의 Cloud Zoo에 추가합니다]].
  - Rhino가 시작할 때 Rhino에 로그인합니다.


=====회사와 학교를 위한 Cloud Zoo 팀=====
Cloud Zoo에서 팀을 생성할 수 있습니다. 팀은 Zoo에서 여러 라이선스를 사용할 수 있는 사용자들로 구성된 그룹입니다.

**Cloud Zoo 팀 설정**

  - [[https://accounts.rhino3d.com/help#create_account|Rhino 계정을 만듭니다]].
  - [[https://accounts.rhino3d.com/help#create_group|나의 Cloud Zoo에 팀을 만듭니다]].
  - [[https://accounts.rhino3d.com/help#licenses|나의 팀에 라이선스를 추가합니다]].
  - 다음 중 하나의 방법으로 나의 팀에 구성원을 초대합니다:
      - [[https://accounts.rhino3d.com/help#invite_group_members|개별적인 구성원을 초대합니다]].
      - [[https://accounts.rhino3d.com/help#link_domain|조직의 도메인을 팀에 연결시킵니다]].

===팀 구성원의 Rhino 사용 방법===

  - 팀 구성원이 초대를 승락합니다.
  - 팀 구성원은 Rhino를 시작할 때 로그인하는 방식으로 Rhino를 사용할 수 있습니다.


**도메인이 연결된 팀**

대규모 팀을 보다 쉽게 관리하기 위해, 도메인을 통해 연결된 계정을 설정할 수 있습니다. 해당 도메인과 일치하는 이메일 주소를 가진 사용자가 자동으로 도메인 연결 팀에 가입하고 라이선스에 액세스할 수 있습니다. 초대 이메일 확인이 필요하지 않습니다. 도메인 연결 팀은 사용자 지정 인증 공급자(Google, Active Directory 또는 One Login)를 지원하여 중앙에서 사용자를 관리할 수 있게 합니다. 팀을 도메인에 연결하세요.

===Cloud Zoo 추가 정보===

  - 라이선스 사용 방식을 다른 방식으로 변경할 수 있으나, 그 과정이 자동은 아닙니다. [[ko:rhino:6:change-license|Rhino 6가 Cloud Zoo를 사용하도록 변경하는 방법]]을 참조하세요.
  - 라이선스 사용에 대해 더욱 자세한 [[ko:rhino:faq:licensing|질문과 대답(FAQ)]]을 참조하세요.
