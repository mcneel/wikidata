====== Mac Rhino 질문과 대답(FAQ) ======

--> Mac Rhino에는 시간 제한이 있습니다. 시간이 만료된 후에는 어떻게 됩니까?#

시간 제한은 모든 분들께서 최신 버전을 테스트하고, 따라서 이미 수정된 문제를 반복해서 보고하시는 일을 방지합니다. 

각각의 Mac Rhino WIP 버전이 만료되기 전에 새로운 WIP 버전을 [[http://www.rhino3d.com/download/wenatchee/5.0/wip|다운로드]]하실 수 있습니다. Mac Rhino는 매번 시작할 때마다 WIP 업데이트를 확인하고 새 버전이 있으면 사용자에게 알려 드립니다. 새 WIP 버전이 업데이트되면 [[http://discourse.mcneel.com/ |Discourse 포럼]]의   [[http://discourse.mcneel.com/category/rhino-for-mac|Rhino for Mac]] 항목에 포스팅됩니다.

<-- 

-->Windows 버전과 데이터 파일이 호환됩니까?#

예. Rhino 5 파일은 동일합니다. Rhino 4 파일 또한 Windows Rhino 4용으로 내보낼 수 (export) 있습니다.

<-- 

-->Macintosh 버전에 2버튼식/ 3버튼식 마우스를 사용할 수 있습니까?#

예. 모든 Macintosh 컴퓨터는 멀티 버튼과 스크롤 휠이 있는 마우스를 지원합니다.  버튼이 1개인 Mac 마우스도 사용하실 수 있습니다. Control 키를 누른 상태에서 단추를 클릭하면, 마우스 오른쪽 단추 클릭처럼 실행됩니다.

<-- 

-->Macintosh 버전에서 Spaceball/SpaceNavigator/SpaceExplorer 를 사용할 수 있습니까?#

현재 SpaceNavigator만 지원됩니다. 

<--
 


-->무엇이 안 됩니까?  왜 "Unknown or unimplemented command"라는 메시지가 표시되나요?#

최대한 Windows Rhino 프로그램이 Macintosh 플랫폼에서 지원되도록 계획하고 있으나 아직 모든 기능이 Macintosh WIP 버전에 구현되지 않았습니다. 새 제품이 어떻게 개발되는지에 대한 안내는 Windows Rhino의 [[http://www.rhino3d.com/nr.htm|소프트웨어 개발 과정]]을 참조하시기 바랍니다. Macintosh 버전 개발도 동일한 개발 과정이 적용됩니다. 현재 [[http://www.rhino3d.com/nr.htm#phase1|1 단계: 개발 중(work-in-progress)]] 단계입니다. 

어떤 작업이 가능하고 어떤 작업이 앞으로 구현되어야 하는지는 [[ko:rhino:mac:whatworks|여기를 참조]]하세요.

<-- 

-->Macintosh 버전에서 영어 외의 언어를 사용할 수 있습니까?#

예. 현재 스페인어, 프랑스어, 독일어, 이탈리아어, 한국어, 중국어, 일본어가 지원됩니다. 이 버전은 WIP (개발 중:Work In Progress) 이므로, 번역 작업이 필요한 부분이 있을 수 있습니다. 

Macintosh Rhino는 Macintosh에 설정된 기본 언어를 사용합니다. OS X 기본 언어와 다른 언어를 사용하려면
  - 어플리케이션 메뉴에서 **Rhinoceros**를 클릭하고 **환경설정(Preferences)**을 클릭합니다.
  - **언어 환경설정** 패널에서 사용하시는 언어를 선택합니다.

<-- 
-->Rhino를 설치 제거하는 방법?#

다음 과정대로 실행하시면 사용자의 컴퓨터에서 Mac Rhino가 완전히 제거됩니다:

  - 어플리케이션 폴더에서 Rhino를 제거합니다.
  - **/Library/Application Support/McNeel** 디렉터리를 제거합니다. 튜토리얼을 다운로드하셨다면 이 디렉터리에 튜토리얼 모델과 매뉴얼이 있습니다. 
  - 홈 디렉터리에서 **~/Library/Application Support/McNeel** 디렉터리를 제거합니다. 이 디렉터리는 Rhino에서 선택적으로 사용되며 여러분의 컴퓨터에 없을 수도 있습니다. 사용하신 Toucan 재질, Python 스크립트, 다른 Mac Rhino 플러그인 등이 포함되어 있을 수 있습니다.
  - 홈 디렉터리에서 **~/Library/Preferences/com.mcneel.rhinoceros.plist** 파일을 제거합니다. 이 파일에 모든 Rhino 설정이 있습니다.

<-- 

-->스크립트와 플러그인을 작성하고 실행할 수 있습니까?#

-->Macintosh 버전에서 명령 스크립트를 실행할 수 있습니까?#

예. 명령 스크립트를 로드하고 실행하려면 Rhino 메뉴에서 **도구 > 명령 > 파일에서 읽기**를 선택합니다.
 
<--

-->Macintosh 버전에서 RhinoScript 프로그램을 실행할 수 있습니까?#

아니요. RhinoScript는 VisualBasic 기반이며, VisualBasic은 OS X에서 실행되지 않습니다. Mac 과 Windows 버전에서 모두 실행되는 스크립트 언어로 Python을 쓰고 있습니다. Python을 사용하시면 모든 Rhino 버전을 위한 스크립트를 작성하실 수 있을 것입니다. Windows와 OS X 모두를 위한 Python 스크립트 지원은  [[http://discourse.mcneel.com/category/scripting | Rhino 스크립팅 포럼]]을 참조하세요.

<--


-->Mac Rhino에서 서드 파티 (third-party) 플러그인이 실행됩니까?#

아마 실행될 것입니다. 그러나, Windows 용 플러그인은 Mac에서 실행되지 않습니다. 플러그인을 Mac Rhino로 포트할지의 여부는 해당 플러그인 개발자가 결정하는 사항입니다.
<--
 
-->제가 만든 C 플러그인이 Macintosh 버전에서 지원됩니까?#

궁극적으로는 지원이 될 것입니다. 그러나, 현재 C 플러그인을 지원할 준비가 되어 있지 않습니다. 앞으로는 지원할 계획을 갖고 있습니다. 여러분이 직접 Macintosh 개발 시스템을 사용하여 해당 플러그인을 다시 컴파일해야 합니다. 플러그인이 얼마나 복잡한 지에 따라 달라지긴 하지만, 작은 작업만으로도 Windows와 Macintosh 양쪽에서 컴파일되고 실행되는 플러그인을 만드실 수 있을 것입니다. 

<--

-->제가 만든 .NET 플러그인이 Macintosh 버전에서 지원됩니까?#

궁극적으로는 지원이 될 것입니다. Rhino 플러그인 개발자들이 OS X와 Windows에서 모두 실행되는 플러그인을 작성할 수 있게 만드는 툴을 당사에서 개발 중입니다.

<-- 
<--

-->Mac Rhino는 언제 출시됩니까?#

아직 미정입니다. Robert McNeel & Associates에서는 미완성인 소프트웨어를 미리 릴리스하고 여러분의 피드백을 바탕으로 개발을 완성하는 독특한 개발 과정을 채택하고 있습니다. Mac Rhino는 [[http://www.rhino3d.com/nr.htm|Windows Rhino 개발]]과 동일한 과정으로 개발되고 있습니다.

업그레이드 정책에 대해서는 아직 아무것도 정해진 바가 없습니다.

-->문제는 어떻게 보고합니까?#

[[http://discourse.mcneel.com/category/rhino-for-mac|Mac 포럼]]에 문제 발생에 대한 글을 올려 주세요. 발견하신 문제가 이전에도 언급된 적이 있는지 검색하실 수 있으며, 여러분의 의견을 추가하시거나, 포럼 위쪽의 **Create Topic**을 클릭하여 새 글을 작성하실 수 있습니다.

스크린 캡처 또는 3dm 파일을 첨부하시려면 **Create a new topic** 창 위에 Upload 단추를 클릭하세요.{{:rhino:mac:9-16-2013_2-11-02_pm.png?nolink|}}.



-->대용량 파일을 저희에게 보내시려면#

저희가 문제를 해결하는 데 도움이 되는 파일이 크기가 작다면 [[http://discourse.mcneel.com/category/rhino-for-mac|Discourse 포럼]] 글에 파일을 첨부해 주세요.\\ \\

파일 크기가 큰 경우

  - [[http://www.rhino3d.com/upload|McNeel 파일 업로드]] 웹 페이지로 갑니다. 
  - 귀하의 정보를 입력하시고 McNeel의 담당자 이메일 **<mactech@mcneel.com>**을 수신자로 지정하신 후, 해당 문제를 상세하게 설명해 주세요.

-->오류 보고를 상세히 작성하는 방법#
여러분께서 발견하신 문제를 저희가 수정하려면 문제를 저희쪽에서 반복할 수 있어야 합니다. 보다 상세하게 오류를 보고해 주시면 저희에게 큰 도움이 됩니다. 여러분께는 너무 당연하게 여겨질 수 있는 부분을 포함하여, 최대한 상세하게 순서를 적어 주세요. 오류 보고에 들어가야 하는 내용:

  - 문제를 다시 발생시키는 방법을 순서대로 작성합니다. 명령을 실행할 경우, 명령을 시작할 때 명령을 입력해서 실행했는지, 메뉴에서 선택했는지 등을 명시해주시기 바랍니다.
  - 각 순서를 실행한 이후 어떤 현상이 발생하는지 알려 주세요.
  - 대신, 원래 어떤 작업을 원하셨는지도 알려 주세요. 
  - 해당 문제가 발생하는 모델 파일.
  - 문제가 발생할 때의 스크린 캡처.
  - 사용하시는 컴퓨터의 사양과 Rhino 버전:
    - Rhino Preferences 패널을 엽니다.
    - **OpenGL** 탭을 선택합니다. 사용하시는 컴퓨터에 대한 설명이 표시됩니다.
    - **Copy to clipboard** 단추를 클릭합니다. 탭의 설정 내용이 클립보드로 복사됩니다.
    - 복사된 컴퓨터 사양 설명을 작성하시는 이메일에 붙여넣습니다.

이 문제가 오직 //본인에게만// 발생하는 문제라고 가정하세요.

-->설치 문제#

-->새 버전을 설치하기 전에 Rhino를 설치 제거해야 합니까?#

아니요. 

<-- 

-->Rhino를 설치할 수 없습니다. 어떻게 해야 합니까?#

다운로드하신 .dmg 파일이 유효한지 확인하세요. 간혹, 다운로드하신 파일이 손상되어 있는 경우가 있습니다. 

  - 디스크 유틸리티 프로그램 **/Applications/Utilities/Disk Utility** 을 시작합니다. 
  - 메뉴에서 **Images > Verify...**를 선택하면 Open File 대화상자가 표시됩니다.
  - 다운로드한 dmg 파일을 선택합니다. 일반적으로 다운로드한 파일은 Downloads 폴더에 있으며 파일 이름은  rhinoceros_wip_YYYYMMDD.dmg (YYYY 연도, MM 월, DD 일) 로 구성되어 있습니다.
  - 다운로드한 파일이 유효하지 않다면  [[http://www.rhino3d.com/download/wenatchee/5.0/wip|다운로드]] 웹페이지를 방문하시고 새로운 다운로드 링크를 요청하시기 바랍니다.

<-- 

-->Rhino 최신 버전을 설치하였으나 만료됐다는 메시지가 표시됩니다. 어떻게 해야 합니까?#

귀하의 컴퓨터에는 2개 이상의 Rhino가 설치되어 있으며, 만료된 버전부터 시작하므로 해당 메시지가 표시됩니다. **모든** Rhino를 제거하셔야 합니다. 

아무 3DM 파일을 두 번 클릭합니다. 해당 파일이 Rhino에서 열리면 어딘가 Rhino가 설치되어 있기 때문입니다. 
  - Dock에서 Rhino 아이콘을 두 번 클릭하고 메뉴에서 **옵션 > 파인더에서 표시(Show in Finder)**를 선택하여 위치를 찾습니다.
  - 찾을 Rhino를 Trash로 드래그합니다. 
  - 더 이상 Rhino가 없을 때, 3DM 파일을 두 번 클릭하면 파일이 TextEdit 에서 열립니다.  이제 모든 Rhino가 제거되었습니다. 
  - 방금 다운로드하신 Rhino 버전을 다시 설치하세요.

<--
 
-->Rhino를 Applications 폴더로 드래그하면 BGHUDAppKit 가 실행 중이어서 설치할 수 없다는 메시지가 표시됩니다. 무슨 뜻이죠?#

이것은 10.7 파인더에 있는 오류입니다. 복사본을 시작하기 전에 어플리케이션을 덮어 쓸 수 있는지 올바르게 확인하지 않습니다. 이 오류 메시지가 표시되거나, Info.plist에 대한 오류 메시지가 표시될 수 있습니다.

안타깝게도 이 메시지가 표시되는 것은 파인더가 기존 Rhino 어플리케이션의 일부를 벌써 삭제하고 새 버전을 설치하지 않았음을 뜻합니다. 즉, 현재 Rhinoceros가 손상되어 다시 실행하려고 하면 크래시가 발생하게 됩니다.

드래그 앤 드롭을 두 세 차례 실행하면 결국엔 실행됩니다. 새 버전을 설치하기 전에 Applications 폴더에 있는 Rhinoceros를 Trash 로 드래그할 수도 있습니다.  이제 파인더가 기존 버전을 덮어 쓰는 데 문제가 없습니다. 
 
10.8 파인더는 이 작업을 올바르게 실행합니다.  10.8로 업그레이드하는 것도 또 다른 해결책입니다.

-->Mac Rhino가 크래시했습니다! 어떻게 해야 합니까?#


Rhino가 크래시하면 돌아가는 공이 잠시 표시되고 다음 메시지가 표시됩니다:

{{:rhino:mac:rhinocrash.png?nolink|}}

이 대화상자의 **Report** 단추를 **클릭하지 마세요**. 이 단추를 클릭하시면 당사가 아니라 Apple에게 오류 보고가 전송되며, 저희는 영원히 볼 수가 없습니다.

**McNeel로 크래시 보고를 보내려면**

  - 이 대화상자의 **Reopen** 단추를 클릭합니다. Rhino가 다시 시작됩니다. Rhino에서 마지막으로 Rhino가 실행됐을 때 크래시가 발생했음을 감지되어 **Rhino Crash Report** 대화상자가 표시됩니다:
  - Rhino에서 크래시가 발생했을 때 어떤 작업을 하고 계셨는지 상세하게 적어 주세요. 저희쪽에서도 해당 문제를 반복할 수 있어야 수정할 수 있습니다. 크래시를 반복할 수 있을 만큼 자세히 적어 주시는 것이 중요합니다. 이메일 메시지가 작성되고 McNeel 개발팀 앞으로 해당 이메일 전송됩니다. 이 이메일 메시지에는 Problems 와 System Information Box에 보이는 정보와 입력하신 정보만이 포함됩니다.
  - **Send to McNeel** 단추를 클릭합니다. Rhino가 종료됩니다.

-->Mac Rhino의 최신 버전에 문제가 있어, 이전 버전을 다운로드하고 싶습니다.#

이전 WIP 버전인 Mac Rhino는 사용하실 수 없습니다. 개발 중인 소프트웨어 테스트 프로그램에 참여하고 계시므로 문제가 발생했을 때 이를 보고해 주시는 것에 동의하셨습니다. [[ko:rhino:mac:reportingproblems|Mac Rhino에서 문제 보고]] 글을 참조하셔서 문제를 보고해 주세요. 저희가 문제를 해결하는 데 큰 도움이 됩니다.

-->"Program Failure"라는 이상한 메시지가 표시되고 난 후, Mac Rhino가 더 이상 실행되지 않습니다.#

Mac Rhino에서 예상하지 못 한 상황에 부딪히면 일반적인 "Program Failure" 메시지가 표시됩니다. 이 오류가 발생한 후에는 Mac Rhino가 내부적으로 종료되고 이 시점 이후 명령 또는 메뉴를 실행하거나 선택할 수 없게 됩니다. 파일을 저장하고 Mac Rhino를 끌 수 있지만, 다른 작업은 불가능합니다. 이 문제가 언제 발생했는지 저희에게 알려 주셔야 합니다. 
Rhino에서 작업하실 때 다음 메시지를 보실 수 있습니다:

몇 가지 상황에서 일반적인 메시지가 Rhino에 표시됩니다. Rhino에 아직 모든 기능이 구현된 것은 아닙니다. 아직 구현되지 않은 기능을 사용하려고 하시면 이 메시지가 표시됩니다. Windows용 Rhino에서만 사용되어야 하는 프로그램 코드도 있습니다. 이 코드를 실행시키는 어떤 작업을 하시게 되면 "Rhino Error" 메시지가 표시됩니다. 마지막으로, 초기 코드에는 많은 디버깅 테스트가 있습니다. 발생해서는 안 되는 상황을 내부 테스트에서 발견하게 되면 이 메시지가 표시됩니다.

{{:rhino:mac:notimpl.png?nolink|}}

이들 메시지를 보시게 되면 해당 메시지에서 파일 이름, 줄 번호, 버전 정보를 기록해 두세요. 이 정보와, 이 메시지가 표시되었을 때 어떤 상황이었는지에 대한 설명을 적어 <mactech@mcneel.com>에게 이메일로 보내주세요. 해당 문제가 저희쪽에서도 발생되는지 확인하고 해결책을 찾아 다음 번 릴리스에 반영시키도록 하겠습니다. 

이 메시지를 받으신 후에는 Rhino에서 몇 가지 명령이 제한될 수 있습니다. 파일을 저장하고 Rhino를 끌 수는 있으나 그 외 많은 작업이 안 될 것입니다.

--> 오른쪽 클릭으로 마지막 실행된 명령을 다시 실행하고 싶습니다 #

상황에 맞는 메뉴(context menu)를 사용 안 함으로 설정하시면 이 작업을 하실 수 있습니다.

  - 어플리케이션 메뉴에서 **Rhinoceros**를 클릭하고 **환경설정(Preferences)**을 클릭합니다.
  - **마우스** 패널을 선택합니다.
  - **마우스 오른쪽 단추** 항목에서 **Enable Context Menus** 확인란을 선택 해제합니다.

<--

--> 도구 단추에서 두 번째 명령을 오른쪽 클릭으로 실행하고 싶습니다#

일부 툴 팔레트 단추에는 해당 단추에 2개의 명령이 있습니다. 명령은 도구설명으로 표시됩니다.
두 번째 명령을 실행하는 기본 방법은 해당 도구 단추를 Option 키와 함께 클릭하는 것입니다. 

  - 어플리케이션 메뉴에서 **Rhinoceros**를 클릭하고 **환경설정(Preferences)**을 클릭합니다.
  - **툴 팔레트** 패널을 선택합니다.
  - **Execute alternate command when** 설정을 지정합니다. 또한, 도구모음과 툴 팔레트는 오른쪽 클릭을 같은 방식으로 처리합니다. 

**안내:** 오른쪽 클릭을 두 번째 명령이 실행되도록 정의했다면 오른쪽 클릭으로 문서 도구모음을 사용자 지정할 수 없습니다. View/Customize Toolbar 메뉴 선택을 대신 사용해야 합니다.

Mac UI 표준에서 오른쪽 클릭(또는 control-클릭)은 //항상// 컨텍스트 메뉴를 엽니다. 이 툴 팔레트 패널에서 원하는 대로 설정을 변경하실 수 있습니다.
