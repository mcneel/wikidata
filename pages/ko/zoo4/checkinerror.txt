======"CD Key addition request denied by the server (CD 키 추가 요청이 서버에서 거부되었습니다)" 오류======
> **제품:** //[[:ko:zoo4:home|Zoo 4.0]]//
> **요약:** //"CD 키 추가 요청이 서버에서 거부되었습니다..." 오류 문제 해결.//

=====질문=====
Zoo 서버에서 라이선스를 받아 사용하는 Rhinoceors가 있습니다. 사용자가 라이선스를 체크아웃하여 사무실 이외의 공간에서 사용할 수 있었습니다. 그러나 해당 사용자가 다시 라이선스를 체크인하려고 하자, 다음 오류가 표시되었습니다:

//CD Key addition request denied by the server. Contact your system administrator.//

{{:zoo:zoo_checkin_failed.jpg|}}

예전에는 항상 실행됐었습니다. 문제 해결 방법대로 따라해봤으나 해결에 도움이 되지 않았습니다.

=====답변=====
이 오류는 다음과 같은 몇 가지 이유로 발생했을 수 있습니다: 

  - 라이선스 체크아웃이 Zoo 서버에서 허용되어 있지 않습니다. Zoo 서버에 라이선스 체크아웃이 허용되어 있지 않으면 라이선스를 체크인할 수 없습니다.
  - 체크아웃된 라이선스가 Zoo 콘솔로부터 수동으로 체크인되었습니다. 따라서, Zoo는 해당 라이선스가 체크아웃되어 있음을 더 이상 알지 못합니다.
  - 시스템이 실제로는 독립실행형(standalone) 라이선스입니다. CheckInLicense를 실행하면 Zoo가 해당 CD 키를 Zoo의 라이선스 풀로 추가하려고 시도하며 해당 라이선스가 이미 존재하는 경우 작업이 실패합니다.

Zoo 콘솔에 가셔서 문제가 발생한 컴퓨터에 라이선스가 여전히 체크아웃되어 있는지 확인하세요. 체크아웃되어 있는 상태가 아니라면, Rhino의 SYSTEM 폴더에 있는 라이선스 키 (**RHINO.KEY**)를 삭제하고 다시 Rhino를 시작하시기 바랍니다.

해당 라이선스가 여전히 Zoo에 체크아웃된 상태라면 [[:zoo:troubleshooting|Zoo 문제 해결]] 안내대로 실행하시고 컴퓨터에 무슨 변화가 있는지 확인하세요.
