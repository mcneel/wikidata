**[[BadObjects|Que faire avec des mauvais objets ?]]**
//J'ai un mauvais (invalide) objet, comment cela est-il arrivé et comment le réparer ?//

Les "mauvais" objets dans Rhino sont des objets qui violent certaines règles NURBS ou qui présentent un problème au niveau de leur structure.

En théorie, Rhino n'est pas sensé produire de mauvais objets, mais en pratique, cela est possible et arrive parfois.

Des mauvais objets peuvent aussi être créés lors de l'importation de fichiers d'autres programmes.

Une fois que vous savez comment les traiter, ce n'est normalement pas très long ni difficile de les réparer - voici quelques conseils.

======Problèmes d'importation/exportation======
=====Pourquoi mon fichier DWG d'AutoCAD est-il vide quand je l'ouvre dans Rhino ?=====
Avant d'ouvrir le fichier, vérifiez que les lignes sont décomposées en éléments de base. En d'autres termes, vérifiez que votre modèle ne contient pas de polylignes. Si cela ne marche pas, utilisez le format DXF au lieu de DWG.

=====Pourquoi ma souris Logitech (curseur) ralentit dans Rhino ?=====
Vérifiez que vous avez les pilotes (Mouseware) 9.76 ou supérieurs.  Ouvrez le logiciel de configuration de la souris.

allez dans l'onglet Mouvement et DÉSACTIVEZ la case "désactiver l'accélération dans les jeux".

Mouseware pense que toute application utilisant **OpenGL** (comme Rhino) est un jeux.

=====Pourquoi ma tablette Wacom ne fonctionne pas correctement ?=====
Les utilisateurs de Rhino 4.0 qui disposent d'une tablette Wacom ont souvent les observations suivantes :

  - Le déplacement par panoramique, zoom ou rotation dans une fenêtre annule le travail sur le bord d'une vue, contrairement à la souris.
  - Le déplacement par panoramique, zoom ou rotation sur le bord d'une vue envoie la vue à l'infini.
La tablette Wacom **doit être configurée en mode souris**et non en mode crayon. Cette contrainte existait également dans les versions précédentes de Rhino.

La différence avec la version 4, c'est que le curseur est enroulé autour de la fenêtre active.

Dans les versions précédentes, le curseur était enroulé autour de tout l'écran. Vous remarquez donc l'effet décrit ci-dessus plus rapidement dans la version 4.

=====Pourquoi mon appareil de type Spaceball ne fonctionne-t-il pas correctement ?=====
**Question** - après avoir réalisé une mise à jour de Rhino 3.0 vers 4.0, j'ai remarqué que mon Space Pilot n'effectue que des rotations. Je ne peux plus configurer mes boutons ni mes vues et le bouton "fit" ne fonctionne plus. J'ai les derniers pilotes de http://www.3dconnexion.com.

**Réponse** - le nom du fichier exécutable a été modifié entre les versions 3 et 4 de Rhino. Si le pilote ne charge pas à votre configuration de Rhino, ouvrez le fichier Rhinoceros.scg qui se trouve dans le dossier en_us/profiles/default du répertoire 3DxWare et tapez **rhino4.exe** comme nom des EXECUTABLES. Redémarrez ensuite le pilote et Rhino.

Le lien vers le dernier pilote répare ce problème : http://www.3dconnexion.com/support/4a1.php

Nous vous recommandons de mettre à jour les pilotes toutes les 8 semaines environ afin d'obtenir toutes les corrections d'erreur et les mises à jour des versions.

**[[FarFromOrigin|Loin de l'origine...]]**
À l'heure actuelle, la géométrie construite dans Rhino qui se trouve très loin de l'origine du repère général présente des problèmes d'affichage et de précision. Ceci est dû aux maillages simple précision et au très grand nombre d'unités impliquées. Une des solutions est de déplacer toute la géométrie plus près de l'origine et réaliser le modèle à cet endroit. Cependant il se peut que le modèle doive être placé loin de l'origine pour être conforme à des données de levés qui utilisent le très grand nombre d'unités. Vous pouvez l'avoir des deux façons en utilisant cette [[FarFromOrigin.|procédure]]

=====L'affichage des lignes n'est pas correct dans les trois vues parallèles avec la mise à jour vers Rhino 4 : Fermer les fenêtres=====
Si vos courbes et vos lignes ne sont pas affichées correctement dans les trois vues parallèles tout en étant correctes dans la vue perspective, commencez par afficher toutes les vues sur l'écran. Cliquez avec le bouton de droite sur le titre de chaque fenêtre parallèle et sélectionnez Fermer la vue dans le menu. Une fois que toutes les fenêtres avec un mauvais affichage sont fermées, allez dans le menu Vue, sélectionnez Disposition des fenêtres et choisissez 4 vues. Les courbes et les lignes devraient apparaître correctement.

<color blue>Ce problème a été résolu dans Rhino V4 SR-3. -John Brock</color>

**[[ATiRadeonCards|Problèmes avec les cartes ATi Radeon]]**
Selon plusieurs utilisateurs, lorsque des pilotes ATi plus récents que la version 8.4 (la version actulelle est 8.10 à l'heure de ce message) sont installés sur les cartes Radeon, les fenêtres de Rhino ne fonctionnent pas correctement.  <color red>Parmi les symptômes on retrouve des retards de 1-2 secondes lors d'un déplacement en panoramique, en zoom ou en rotation.</color>  cette solution a été testée sur les cartes HD, cependant elle semble ne pas fonctionner sur les cartes [[rhino:firegl|FireGL]].

Ajouté le 16/10/2008 -John Brock
